# AnnealMusic API & Schema Stability Commitments: Version 8 Closeout

To guarantee that research projects, clinical trials, and data sonifications initiated across the historical development cycles remain fully operational and reproducible for years, AnnealMusic establishes these formal **API and Schema Stability Commitments**.

All components documented below are frozen and backwards-compatible. Any future updates must respect these interfaces or provide documented, automatic translation scripts.

---

## 1. Collaborative Bridge Transport (v5.0)

- **Bridge v1 Format**: The collaborative bridge transport protocols and structural payload signatures are frozen and stable indefinitely.
- **Backward-Compat Tolerances**: All collaborative session synchronization signals must parse cleanly, enabling legacy client versions to link sessions seamlessly.

---

## 2. Open Sound Control (OSC) Namespace (v5.1)

- **OSC Namespace v1**: The bidirectional OSC address space (`/anneal/state/*` and `/anneal/engine/*`) is locked and **additive-only**.
- **Control Targets**: Slider controls and mapping templates (`/control/slider/*`) maintain stable mappings to prevent interface failures for external performers.

---

## 3. Command Line Interface (CLI) Surface (v5.2)

- **CLI Surface Surface**: The command-line flags, options, and commands are strictly **additive-only** within major versions to prevent breakage in automated scripting harnesses and lab servers.
- **Audio Output Parity**: Raw audio generated via headless CLI commands must maintain amplitude variance within **0.001%** compared to historical master reference recordings.

---

## 4. High-Performance Session Datalogger (v5.3)

- **Datalogger Schema v1**: The serialization outputs (JSON Lines, Tabular CSV columns, HDF5, Parquet) are stable indefinitely.
- **Temporal Alignment**: Tabular tick metrics generated by the flattener preserve their column headers, guaranteeing direct ingestion by downstream Pandas and PyArrow converters.

---

## 5. Scientific Python Scripting Sandbox (v5.4 – v5.6)

- **`anneal` Module**: The Python scripting environment's core namespaces (`state.get()`, `state.set()`, `engine.get_spectrum()`, and `datalog.start()`) are frozen and strictly additive.
- **SciPy / Pandas Integration**: Numerical script execution in Pyodide Web Workers maintains deterministic VFS path resolution for loading external datasets.

---

## 6. Study Snapshot & Publication DOI Permanence (v7.0)

- **Zenodo DOI Permanence**: Snapshots of clinical protocols and resources published against the Zenodo API are guaranteed permanent, immutable records.
- **Clinical Schema**: The database tables mapping studies (`studies`, `study_investigators`, `study_audit_log`) support only backwards-compatible nullable column upgrades.

---

## 7. URL Parameter Schema Compatibility

- **URL Schema Compatibility**: Every generated URL parameter schema version (from `v1` through **`v22`**) is supported by the client-side hydrator.
- **Parameter Drift**: Minor visual and audio parameter drift is bounded and resolved gracefully, ensuring legacy sharing configurations load correctly.
