# Implementation Plan — v4.4 Breath Pacing Visuals

This plan details the architecture for **v4.4 Breath Pacing Visuals**: an optional,
silent, visual-only breath-pacing overlay that composes on top of the existing
v1.9 visualizer during Listening Sessions, Drone Mode, and the v4.3 Standalone
Meditation Timer. There is **no audio** in this feature — the music remains the
only audio surface. Users opt in per session (Listening) or per device (Drone).

> **Branch:** `feat/v4.4-breath-pacing` (development on `claude/ecstatic-einstein-I8mAs`)
> **Prereq:** v4.3 shipped (bells library + standalone breathing timer live).

---

## 1. Pattern Data Model + URL v20

### 1.1 The 4-tuple

A breath pattern is four durations in seconds, in cycle order:

```ts
// src/breath/patterns.ts
export type BreathTuple = [
  inhale_s: number,
  hold_full_s: number, // 0 = no hold at peak
  exhale_s: number,
  hold_empty_s: number, // 0 = no hold at trough
];

export type BreathPatternId =
  | 'box'
  | '4-7-8'
  | 'coherent'
  | 'resonance'
  | 'custom';

export interface BreathPattern {
  id: BreathPatternId;
  tuple: BreathTuple; // for 'custom', filled from custom_pattern
  custom_pattern?: BreathTuple;
}
```

### 1.2 Built-in patterns

| id          | label                         | tuple              | notes                       |
| ----------- | ----------------------------- | ------------------ | --------------------------- |
| `box`       | Box Breathing (4-4-4-4)       | `[4, 4, 4, 4]`     | Symmetric square cycle.     |
| `4-7-8`     | 4-7-8 Breathing               | `[4, 7, 8, 0]`     | Popularized by Andrew Weil. |
| `coherent`  | Coherent Breathing (5.5/min)  | `[5.5, 0, 5.5, 0]` | ~5.5 breaths/min.           |
| `resonance` | Resonance Breathing (4.5/min) | `[6, 0, 6.5, 0]`   | HeartMath-style.            |
| `custom`    | Custom                        | user-entered       | No framing copy (§4).       |

Built-ins live in one frozen registry so durations are declared exactly once.

### 1.3 URL schema v20

`src/share/schema.ts`:

- `SCHEMA_VERSION = 20`.
- Append `20` to `SUPPORTED_SCHEMA_VERSIONS` (v1–v19 still decode; a v20 doc with
  no breath field decodes identically to v19).

The breath field rides on the **Listening Session** payload only (Drone breath is
device-local, not URL-shared — see §C2). Encoding format (compact, matches existing
`key=value&...` style in `encode.ts`):

```
&br=<id>                       // present only when a pattern is attached
&br_c=<i>,<hf>,<e>,<he>        // present only when id === 'custom'
```

`DecodedListeningSession` gains an optional field:

```ts
breath_pattern?: {
  pattern: 'box' | '4-7-8' | 'coherent' | 'resonance' | 'custom';
  custom_pattern?: [number, number, number, number];
} | null;
```

Decode is tolerant: unknown id → drop to `null` (no overlay) with a warning, never
throws. Custom tuple is clamped to bounds (§7) on decode.

### 1.4 Persistence surfaces

| Surface           | Storage                                           |
| ----------------- | ------------------------------------------------- |
| Listening Session | `breath_pattern` JSONB column + URL v20 field     |
| Drone Mode        | `localStorage['am_breath_drone']` (device-scoped) |
| Standalone Timer  | `localStorage['am_breath_timer']` (device-scoped) |

DB migration (refined from the prompt's sketch):

```sql
ALTER TABLE listening_sessions
  ADD COLUMN breath_pattern JSONB;   -- nullable, no default, no backfill
```

NULL means "no breath overlay" — semantically identical to a pre-v4.4 row.

## 2. Breath Visual Design (size easing, phase indicator, palette)

### 2.1 The circle

A single slow-pulsing circle centered in the visualizer.

- **Radius** interpolates between `rMin` and `rMax` (fractions of `min(w,h)`):
  `rMin ≈ 0.10`, `rMax ≈ 0.30`. Tunable constants in `breath/patterns.ts`
  (or a small `breath/visualConstants.ts`) — never inline.
- **Inhale:** radius eases `rMin → rMax` with **ease-in-out** (`smoothstep`),
  so the start and end of the breath are gentle, not linear.
- **Hold (full):** radius pinned at `rMax`. Optional faint `"hold"` label
  (lowercase, low-contrast) — **no numerals** ever.
- **Exhale:** radius eases `rMax → rMin` (ease-in-out).
- **Hold (empty):** radius pinned at `rMin`.

### 2.2 Phase-indicator ring

A thin ring around the circle whose **sweep angle** (0 → 2π) tracks progress
through the _entire_ cycle (not per-phase), giving a calm "where am I in the
breath" sense without numbers. Drawn as an arc stroke, ~1.5px, low alpha.

### 2.3 Palette (warm amber, reuse `PALETTE`)

Reuse existing tokens from `src/visual/canvas/palette.ts` — do **not** introduce
a new color family:

- Circle fill: `PALETTE.glowCore(a)` / `glowMid(a)` radial, amber.
- Phase ring: a low-alpha amber derived from `loopRing`/`inputRing` amber `[245,158,11]`.
- New constants (radii, line widths, label alpha) go in a `BREATH` block alongside
  `VISUAL`, keeping the breath overlay's tuning in one obvious place.

### 2.4 No counting numbers

Per the contemplative-session constraint: the only optional text is a faint
lowercase `"hold"` during hold phases. No seconds, no counts, no "breathe in".

## 3. Visualizer Composition Strategy

The overlay must feel **part of** the visualizer, not a sticker on top.

- **Rendering layer:** The breath circle draws into the _same_ canvas region,
  composited as an overlay pass. Because the existing renderer is a
  `VisualRenderer` (`drawFrame(state, now)`), the cleanest seam is to render the
  breath overlay as a **sibling absolutely-positioned `<canvas>`** owned by
  `BreathOverlay.tsx`, stacked above the visualizer canvas with the same
  bounds. This keeps the v1.9 renderer untouched (low risk) while letting the
  breath layer use its own RAF driven by `BreathController`.
- **Brightness dip:** When breath is active, the underlying visualizer dims
  slightly so the circle reads. Implemented via a CSS `opacity`/`filter:brightness`
  on the visualizer canvas (e.g. `0.82`) transitioned smoothly on enable/disable —
  no change to renderer internals, fully reversible.
- **Sizing:** `BreathOverlay` reads the visualizer's measured size (shared
  ref/ResizeObserver) so center and radius scale identically.

Rationale for the sibling-canvas approach over threading breath into `VisualState`:
it keeps phase math out of the hot DSP-visual path, avoids touching the
WebGL/Canvas renderers (two implementations), and makes reduced-motion / haptics
local concerns of the overlay. The two canvases share size only.

## 5. Mobile Haptic Integration (Capacitor `@capacitor/haptics`)

- Add `@capacitor/haptics@^8` (matches our `@capacitor/*` 8.3.x).
- `src/breath/hapticBridge.ts` wraps it:
  - `isHapticsAvailable()` — true only on native (Capacitor) platforms.
  - `pulsePhaseTransition(phase)` — fires a _light_ `Haptics.impact({ style: Light })`
    at each phase boundary. Web → no-op. Import is dynamic/guarded so web bundles
    don't pull native code.
- **Off by default.** Controlled by a `haptics` boolean pref (device-local,
  `am_breath_haptics`). Only meaningful on mobile; the toggle is hidden/disabled on web.
- `BreathController` exposes phase-transition events; `BreathOverlay` (or a small
  effect) calls `pulsePhaseTransition` on each transition when the pref is on and
  native is available. Phase math stays in the controller (single source).

## 6. `prefers-reduced-motion` Handling

Precedent: `src/components/InfoTip.tsx` already suppresses animation under
`prefers-reduced-motion`. We follow the same approach.

- **Source of truth:** OS-level `window.matchMedia('(prefers-reduced-motion: reduce)')`,
  **plus** an in-app "Reduce motion" pref (`am_breath_reduce_motion`) that can force
  it on independently of the OS setting. Effective = OS-reduce **OR** app-pref.
- **Reduced behavior:** No size pulse. The circle holds a fixed radius and instead
  **color-shifts / fades** between phases (amber alpha cross-fade: brighter on
  inhale, dimmer on exhale). The phase ring still sweeps (it is slow, not a pulse).
- **Haptics:** unaffected by reduced-motion (they're non-visual); they remain
  available and still off-by-default.
- Implemented once in `BreathOverlay` by branching the draw on an
  `effectiveReducedMotion` boolean derived from the controller's phase fraction.

## 7. Custom Pattern UI (4 number inputs with sensible bounds)

Four numeric inputs: Inhale · Hold (full) · Exhale · Hold (empty).

Bounds (single `clampCustomPattern()` in `patterns.ts`, reused by UI + decode):

- `inhale_s ≥ 1` (a breath needs an inhale).
- `exhale_s ≥ 1`.
- Each of the four ≥ 0, ≤ 30.
- Holds (`hold_full`, `hold_empty`) may be 0.
- **Total cycle ≤ 60s** (sum of all four). Reject/clamp on exceed with an inline
  hint ("Total cycle must be 60s or less").
- Step `0.5s` (supports 5.5 / 6.5 style values).

UI shows the note "Any breath pattern that feels comfortable is good." and **no**
evidence framing (the user is on their own, per spec). Live preview of the circle
optional in CP3.

### §A — `BreathController` contract (phase math lives once)

```ts
export type BreathPhase = 'inhale' | 'hold-full' | 'exhale' | 'hold-empty';

export interface BreathFrame {
  phase: BreathPhase;
  /** 0..1 progress within the current phase. */
  phaseProgress: number;
  /** 0..1 normalized radius (0 = trough, 1 = peak), eased for inhale/exhale. */
  amplitude: number;
  /** 0..1 progress through the whole cycle (drives the phase ring). */
  cycleProgress: number;
  /** True only on the tick a phase boundary was crossed (drives haptics). */
  transition: boolean;
}

class BreathController {
  constructor(tuple: BreathTuple);
  setTuple(tuple: BreathTuple): void;
  /** `nowSec` is an absolute clock (AudioContext.currentTime in app; injectable in tests). */
  frameAt(nowSec: number): BreathFrame;
  reset(nowSec: number): void;
}
```

Zero-duration phases (holds = 0) are skipped cleanly: the effective cycle length is
the sum of non-zero phases and the controller maps `nowSec` into it. Pure and
deterministic — no internal `Date.now()`/RAF, so it's fully unit-testable and
immune to RAF pauses (see §9 / clock choice below).

## 9. Risks & Mitigations

1. **Distraction over centering.** A pulsing circle could pull attention rather
   than settle it. _Mitigation:_ slow easing (ease-in-out), low-contrast amber that
   matches the palette, brightness dip so it reads without being loud, no numerals,
   opt-in only. Reviewed against the contemplative thesis in CP3.

2. **Cycle drift on backgrounded tabs (RAF pauses).** `requestAnimationFrame` and
   `performance.now()` stall or throttle when a tab is hidden — the existing v4.3
   timer LFO uses wall-clock deltas and _would_ drift. _Mitigation:_ drive
   `BreathController.frameAt(now)` from **`AudioContext.currentTime`**, which keeps
   advancing while audio plays, so when the tab refocuses the breath resumes at the
   correct phase. The orchestrator already exposes its `AudioContext`
   (`getRecordingTap()` returns `{ ctx }`); we add a tiny `getAudioTime()` accessor
   rather than reach through. The Standalone Timer already owns an `AudioContext`
   (`audioCtxRef`) we reuse. RAF is used only to _paint_; the _phase_ is a pure
   function of audio time. Explicitly covered by the CP1 backgrounded-tab test.

3. **Long-session accuracy (30+ min).** Float accumulation avoided — phase is
   `mod(audioTime, cycleLen)`, not an accumulator, so there is no integration drift.
   Covered by the CP3 smoke matrix.

4. **Two renderer backends (Canvas + WebGL).** Avoided entirely by the
   sibling-canvas overlay (§3): neither backend is modified.
