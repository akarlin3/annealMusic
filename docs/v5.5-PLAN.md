# AnnealMusic v5.5 · Scientific Python Integration · Plan

This document outlines the architecture, data modeling, sandboxing strategy, and integration roadmap for introducing scientific Python libraries (**SciPy**, **pandas**, **matplotlib**) into the browser-sandboxed Pyodide environment.

## 1. Final Package Whitelist, Sizes & Load Estimates

To maintain the performance standards of AnnealMusic, we establish a strict package whitelist. Below are the estimated sizes (uncompressed / gzipped) and first-time load estimates (assuming standard 50Mbps broadband connection) when fetching from the JSDelivr/Pyodide CDN.

| Package          | Pyodide Package Name | Size (Compressed) | Size (Uncompressed) | Load Time (Cold) | Load Time (Cached) |
| :--------------- | :------------------- | :---------------- | :------------------ | :--------------- | :----------------- |
| **NumPy**        | `numpy`              | ~5.1 MB           | ~15.2 MB            | ~1.5 - 3.0 sec   | < 100 ms           |
| **SciPy**        | `scipy`              | ~16.8 MB          | ~54.6 MB            | ~4.0 - 9.0 sec   | < 100 ms           |
| **matplotlib**   | `matplotlib`         | ~3.2 MB           | ~9.8 MB             | ~1.0 - 2.5 sec   | < 100 ms           |
| **pandas**       | `pandas`             | ~6.5 MB           | ~21.2 MB            | ~2.0 - 4.5 sec   | < 100 ms           |
| **scikit-learn** | `scikit-learn`       | ~8.2 MB           | ~26.4 MB            | ~2.5 - 5.5 sec   | < 100 ms           |
| **PyArrow**      | `pyarrow`            | ~35 - 40 MB       | ~120+ MB            | _Flagged_        | _Flagged_          |

### PyArrow / Parquet Risk Assessment

> [!WARNING]
> Pyodide `0.26.4` does not support PyArrow or direct native Parquet out-of-the-box due to complex, heavy C++ compilation dependencies. Even in Pyodide `0.27.x`, PyArrow adds over **35MB** of gzipped WASM payload.
> **Mitigation:**
>
> - Inside the sandboxed Web Worker, we will rely on **pandas JSONL / CSV serialization** or a highly efficient, lightweight **pure-Python Parquet reader/writer** (such as a pure-Python subset) if Parquet file reading is explicitly done inside Python, or simply export the DataFrame as a JSONL/CSV, which our existing robust CLI/backend converter utility compiles to standard Parquet/HDF5 formats.
> - This keeps the browser bundle and memory footprints optimal while maintaining perfect scientific data integrity.

## 2. matplotlib WebAgg-Style Backend Architecture

Because the Pyodide interpreter runs entirely within a background **Web Worker** (`pyodide-worker.js`), it has **no access to the DOM, window, or HTMLCanvasElement**. To render matplotlib plots interactively, we design a custom **virtual Web Worker WebAgg bridge**:

```
[Pyodide Worker Thread]                         [Main UI Thread]
  |                                               |
  |-- 1. User calls plt.show()                    |
  |-- 2. Custom show() intercepts figure          |
  |-- 3. Renders to PNG bytes (Agg backend)       |
  |-- 4. postMessage({ type: 'plot-render' }) --->|
  |                                               |-- 5. Displays in tabbed UI
  |                                               |-- 6. User clicks Zoom/Pan
  |                                               |      on the plot canvas
  |<-- 7. postMessage({ type: 'plot-event' }) ----|
  |-- 8. Python recalculates axes limits          |
  |-- 9. Re-renders figure to PNG                 |
  |-- 10. postMessage({ type: 'plot-render' }) -->|
                                                  |-- 11. Refreshes plot area
```

### Key Implementation Details:

1. **Force 'Agg' Backend:** We configure matplotlib to use the headless `'Agg'` backend via `matplotlib.use('Agg')`.
2. **Monkeypatched `plt.show()`:**
   - Captures the current active figure (`plt.gcf()`).
   - Saves it to an in-memory `io.BytesIO` buffer as a high-density PNG.
   - Posts the raw image bytes to the UI thread.
   - Automatically closes the figure (`plt.close(fig)`) to prevent browser memory leaks.
3. **Interactive Control Bridge:**
   - The UI displays the rendered PNG.
   - Click/drag operations capture normalized coordinate bounding boxes and send them back to the Python environment.
   - Python applies the new bounds: `ax.set_xlim(x1, x2)`, `ax.set_ylim(y1, y2)`, re-renders, and pushes the updated image.

## 3. Extended `anneal` Python Module API

We extend the sandboxed `anneal` module built in `annealPyModule.ts` with five high-value scientific methods:

### 1. `anneal.session_log()`

```python
def session_log(last_seconds=None, format='standard') -> pd.DataFrame
```

- Ingests ticks directly from the circular log buffer.
- Employs `pandas.json_normalize` to flatten nested JSON tick payloads into a clean tabular DataFrame with columns like `params.rootFreq`, `features.rms`, and `drift.orderParameter`.
- Filters records within the `last_seconds` window based on audio timestamps.

### 2. `anneal.stream_log()`

```python
async def stream_log(every_n_ticks=10) -> AsyncGenerator[pd.DataFrame]
```

- Implemented as an **async generator**.
- Uses `asyncio.sleep` to periodically yield control back to the worker event loop, allowing BroadcastChannel message handlers to process new datalog ticks, yielding accumulated logs as flat pandas DataFrames.

### 3. `anneal.sweep()`

```python
async def sweep(params_grid: dict, duration: float = 5.0) -> pd.DataFrame
```

- Asynchronously sweeps a multi-dimensional parameter grid (e.g. `{"drift": [0.1, 0.5], "brightness": [0.3, 0.7]}`).
- Settles for each combination, captures high-resolution datalog features over `duration` seconds, computes average/max performance metrics (mean RMS, max RMS, spectral centroid, spectral flux, zero-crossing rate, mean coupling order parameter), and returns a consolidated summary DataFrame.

### 4. `anneal.features()`

```python
def features(start: float = None, end: float = None) -> pd.DataFrame
```

- Returns a flat DataFrame containing strictly audio features (`rms`, `spectralCentroid`, `spectralFlux`, `zcr`) indexed by session timestamp over the specified interval.

### 5. `anneal.render()`

```python
def render(duration: float = 30.0, format: str = 'numpy', path: str = None)
```

- Standardizes offline rendering:
  - If `format='numpy'`: makes an asynchronous bridge call to render the current patch on the main thread's `OfflineAudioContext`, returning a 2D float NumPy array `(channels, samples)`.
  - If `format='wav'`: renders, encodes to WAV binary format using the frontend's native `encodeWavBuffer` utility, sends the bytes to the worker, and writes them to Pyodide's MEMFS virtual filesystem at `path`.

## 4. Pyodide Virtual File System (VFS) Workflow

Pyodide's virtual in-memory filesystem (`MEMFS`) provides an isolated container for file read/write operations.

- **VFS File Persistence:** Files written to the VFS (e.g., via `anneal.render(duration=30, format='wav', path='/tmp/render.wav')`) are saved in memory inside the worker. They persist for the duration of the browser tab session.
- **Save to Disk Panel UI:**
  - A clean, beautiful panel in the scripting area lists all files in the designated VFS directory (`/tmp`).
  - Queries the worker using a standard message `{"type": "vfs-list"}` which walks the directory using Pyodide's FS API (`pyodide.FS.readdir`, `pyodide.FS.stat`).
  - Provides two actions:
    1. **Download:** Sends `{"type": "vfs-read", "path": "..."}`. The worker returns the raw `Uint8Array` bytes. The UI thread instantiates a browser download using a temporary `URL.createObjectURL(new Blob([bytes], { type: 'audio/wav' }))`.
    2. **Delete:** Sends `{"type": "vfs-delete", "path": "..."}`. The worker calls `pyodide.FS.unlink(path)` to clean up memory.

## 5. Detailed Loading UX

To provide an exceptional visual experience matching the AnnealMusic aesthetic, we establish the following loading flows:

1. **Persistent Scientific preloader Toggle:**
   - A modern toggle switch labeled `"Preload Scientific Environment"` is added to the scripting settings.
   - Persists state per device using `localStorage.getItem('am_scientific_env_preload')`.
   - If enabled, the page starts loading SciPy, matplotlib, and pandas immediately upon initializing Pyodide.
2. **One-Time Load Progress UI:**
   - Shows a sleek, responsive circular or linear indicator detailing the load progress of each package (e.g., _"Loading SciPy [65%]..."_).
3. **AST-Based Auto-Import Scan (Magic Loader):**
   - If the environment wasn't preloaded, when the user clicks **Run**, we use a lightweight Python AST parser to pre-scan the script for imports of `scipy`, `pandas`, `matplotlib`, or `sklearn`.
   - If any are present but not yet loaded, we intercept execution, display a small console message (_"Auto-loading pandas from CDN..."_), load the package dynamically using `await pyodide.loadPackage(pkg)`, and then cleanly resume script execution!

## 6. Performance Budgets

To keep the application highly responsive and stable:

- **Initial UI Payload:** **Zero additional bytes** loaded on initial page hit. Pyodide and scientific packages are only loaded when explicitly toggled or on the first script run.
- **Initialization Target:**
  - Standard NumPy-only runtime: **~2.5 seconds** (on warm cache).
  - Full Scientific runtime: **~6–12 seconds** (on warm cache, local disk load).
- **Execution Overhead:**
  - Memory ceiling: **250MB** inside the Web Worker. Circular buffer checks throw warnings if data logs exceed safety levels.
  - Streaming rate: Datalog updates stream to Python at `50Hz`, yielding minimal latency.

## 8. Risks & Mitigations

1. **CDN Outages / High Latency:**
   - _Risk:_ Heavy packages (like SciPy) might timeout or fail to download on flaky networks.
   - _Mitigation:_ Graceful fallbacks. If the CDN fetch fails, we display an elegant inline alert with detailed instructions and a manual retry button, preserving the main thread.
2. **Worker Memory Bloat during Long Logging Sessions:**
   - _Risk:_ Keeping unlimited ticks in `localTicks` for pandas ingest could cause the worker to crash.
   - _Mitigation:_ Implement a firm capacity limit of **100,000 records** (~20MB memory footprint) inside the worker's circular buffer, matching the main thread's memory limit.
3. **Matplotlib Rendering Lag:**
   - _Risk:_ Heavy plots can freeze UI transitions.
   - _Mitigation:_ Perform all heavy rendering asynchronously on the background worker thread. The UI thread only receives flat image data, keeping UI transitions 100% fluid.
