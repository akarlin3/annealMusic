# Technical Specification & Implementation Plan — v3.2 Notation Overlay

This document details the architecture, data models, UI designs, audio player integration, and migration plan for introducing a monophonic **notation track** to AnnealMusic.

---

## 1. Notation Data Model & URL v10 Schema

### Data Model

A notation track is represented as a single-voice, monophonic melody line on a `Piece`. We introduce the `NotationNote` interface and add the optional `notation` array to the `Piece` interface:

```typescript
export interface NotationNote {
  id: string; // CRDT-friendly unique identifier (e.g., UUID or short stable random ID)
  onset_ms: number; // Onset time of the note in milliseconds from the start of the piece
  duration_ms: number; // Duration of the note in milliseconds
  pitch_midi: number; // MIDI pitch number (C0 [12] to C8 [108])
}
```

On the `Piece` interface (`src/piece/types.ts`):

```typescript
export interface Piece {
  // ... existing fields ...
  notation?: NotationNote[];
}
```

### URL Schema Bumps to v10

To support sharing pieces that contain a custom notation track, we bump the `SCHEMA_VERSION` in `src/share/schema.ts` to `10`.

We serialize the `notation` array as a compact query parameter to avoid URL length blowup:

- **Format:** `notation=<onset_ms>,<duration_ms>,<pitch_midi>;<onset_ms>,<duration_ms>,<pitch_midi>;...`
- **Example:** `notation=0,1500,60;2000,1000,64;3000,2000,67`
- **Decode Behavior:** When loading a URL, if the `notation` parameter is present, we split by `;` and `,` to reconstruct the `NotationNote` array. Since note IDs are purely for local React/CRDT tracking and are immutable, we generate new unique IDs (e.g., `note-${index}-${random_suffix}`) upon decode.

**Backward Compatibility:**

- Version <= 9 pieces are loaded with `notation = []`.
- Old URLs decode perfectly with fallback defaults.

## 2. Piano-Roll Editor Design & Interactions

The notation editor will be presented as an expandable/toggleable overlay or tab on the timeline editor, providing first-class horizontal and vertical scroll + zoom.

```
+-----------------------------------------------------------------------------------+
| Notation Piano-Roll Editor                                       [Zoom H: -o- ]  |
| Grid Snap: [ 1/16 v ]                                            [Zoom V: -o- ]  |
+---------+-------------------------------------------------------------------------+
| C8      |                                                                         |
| B7      |      +=================+                                                |
| A#7     |      | Note: E5        |                                                |
| A7      |      +=================+                                                |
| ...     |                                                                         |
| C4      |                                +========================+               |
| B3      |                                | Note: G3 (drag/resize) |-->            |
| ...     |                                +========================+               |
| C0      |                                                                         |
+---------+-------------------------------------------------------------------------+
| Scroll  | <============================ Scroll H ===============================> |
+---------+-------------------------------------------------------------------------+
```

### Key Interactions:

1. **Adding a Note:** Click on an empty cell in the grid to create a note at that pitch and onset.
2. **Moving a Note:** Drag a note block horizontally to change its onset, or vertically to transpose its pitch.
3. **Resizing a Note:** Hover over the right edge of a note block to reveal a resize cursor. Drag it to adjust the note's duration.
4. **Deleting a Note:** Double-click the note, or click the note to select it and press the delete icon (or `BackSpace`/`Delete` keys).
5. **Zoom & Scroll:** Dedicated slider controls for:
   - **Horizontal Zoom:** Modifies the horizontal pixels-per-second ratio (`PX_PER_SEC` equivalent) just for the piano roll.
   - **Vertical Zoom:** Modifies the height of each pitch row.
   - Standard scrollbars allow independent scroll behavior.

## 3. Quantization Specifics

When `tempoBpm` is configured on the piece, the notation editor grid snaps to subdivisions.

- **Configurable Grid Subdivisions:** `1/4` (quarter notes), `1/8` (eighths), `1/16` (sixteenths), `1/32` (thirty-seconds), or `off` (free-time).
- **Quantization Math:**
  - Let $T_{\text{beat}} = 60 / \text{tempoBpm}$ (seconds per beat).
  - Let the subdivision factor $F$ be `1` (1/4), `2` (1/8), `4` (1/16), `8` (1/32).
  - Subdivision duration in milliseconds:
    $$T_{\text{sub\_ms}} = \frac{T_{\text{beat}} \times 1000}{F}$$
  - Quantize both **onset** and **duration** to the nearest grid step:
    $$\text{snappedMs} = \text{round}\left( \frac{\text{valueMs}}{T_{\text{sub\_ms}}} \right) \times T_{\text{sub\_ms}}$$
- When `tempoBpm` is `null` (tempoless piece), snapping is automatically bypassed.

## 4. MIDI Import & Export

We will use the highly robust `@tonejs/midi` library to parse and write standard MIDI files (.mid).

### MIDI Import Flow:

1. **File Select / Drag-and-Drop:** User drops or selects a `.mid` file.
2. **ArrayBuffer Reading:** Read the file as an `ArrayBuffer` and parse it via `@tonejs/midi`.
3. **Multi-Track Resolver:**
   - If the file is monophonic / single-track: Import notes directly.
   - If multi-track: Prompt the user with a clean dropdown dialog to select which track to import as the notation line.
4. **Tempo Match Option:** If the MIDI file defines a tempo, offer a dialog prompt: _"This MIDI file specifies a tempo of [BPM] BPM. Would you like to set the piece tempo to match?"_
5. **Quantization Filter:** Offer a toggle to:
   - Preserve raw micro-timing.
   - Quantize import timing to the nearest grid subdivision.

### MIDI Export Flow:

1. Create a Type-0 SMF (Single-track MIDI file) using `@tonejs/midi`.
2. Write the piece's tempo if `tempoBpm` is set.
3. Write all notes in the `notation` array into the MIDI track. Convert milliseconds to seconds (`time_seconds = onset_ms / 1000`, `duration_seconds = duration_ms / 1000`).
4. Generate the binary buffer, wrap it in a `Blob`, and trigger a browser download dialog (e.g. `piece-title.mid`).

## 5. Playback Integration with Engine Root

During playback, the piece player coordinates the active engine's `rootFreq` updates.

### Sub-Player Engine Scheduling (`NotationPlayer`):

1. **Global Playhead Tracking:** In `PiecePlayer`, we compute the piece-level playhead position:
   $$\text{playheadGlobalMs} = \left( \sum_{i=0}^{\text{activeSegIdx}-1} \text{durationMs}(i) \right) + \text{playheadMs}$$
2. **Active Note Scan:** On each player tick (every 50ms), we query if `playheadGlobalMs` lies within the bounds of any note in `piece.notation`:
   $$\text{onset\_ms} \le \text{playheadGlobalMs} < \text{onset\_ms} + \text{duration\_ms}$$
3. **Note Activation & Overriding:**
   - **Case A (Note is Active):** The active note's pitch is converted to Hz:
     $$f = 440 \times 2^{\frac{\text{pitch\_midi} - 69}{12}}$$
     This frequency overrides the segment-driven frequency. We push this frequency to `Orchestrator.setSharedParams({ rootFreq: f })` and remember it as `lastNotationPitchHz = f`.
   - **Case B (No Active Note):**
     - If the segment actively modulates or overrides root frequency (e.g., an arc segment running, a transition segment interpolating, or a fixed segment with custom `rootFreq` config): **The segment wins!** We pass the segment's calculated frequency.
     - If the segment does NOT actively modulate root frequency (i.e. it relies on piece default parameters): **Notation persists!** We maintain `lastNotationPitchHz` to sustain the pitch between notes.
4. **Transitions (Smooth vs. Instant Change):**
   - Provide a toggle in settings: "Smooth Pitch: [On | Off]".
   - When **Smooth** is enabled, frequency changes are ramped via Web Audio's `setTargetAtTime` using the engine's standard smoothing constant.
   - When **Instant** is enabled, we schedule instant frequency jumps (`setValueAtTime`) to mimic standard keyboard or synthesizer play.

## 6. Risks & Mitigation Plan

| Risk                                                             | Mitigation                                                                                                                                                                                 |
| :--------------------------------------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **URL length limits (browsers/CDNs) with long notation tracks.** | Implement a safeguard that warns the user if their notation track exceeds 100 notes, encouraging them to keep ambient melodies concise, and cap URL generation to avoid failures.          |
| **Monophonic overlapping notes on import/drag.**                 | Validate note placement on import and edit. If notes overlap, apply last-note-priority or auto-truncate the preceding note's duration so that only one voice is active at any millisecond. |
| **External dependency size and bundle size constraints.**        | Bundle `@tonejs/midi` dynamically or only import it on demand when import/export actions are clicked, keeping the main bundle lightweight.                                                 |
