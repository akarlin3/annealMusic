# AnnealMusic v4.6 Plan: Health Integrations + Accessibility + v4 Closeout

This document outlines the engineering specifications, native interface designs, a11y criteria, module layouts, and risk analysis for the v4.6 milestone of AnnealMusic.

---

## 1. Custom Capacitor Plugin Specifications (`HealthBridge`)

### Method Signatures

We declare a unified custom Capacitor plugin named `HealthBridge`.

#### TypeScript/JavaScript Interface

```typescript
export interface HealthBridgePlugin {
  requestPermission(): Promise<{ granted: boolean }>;
  logMindfulSession(options: {
    startDate: string;
    endDate: string;
  }): Promise<{ success: boolean }>;
}
```

### iOS (HealthKit) Implementation Detail

- **Target Category**: `HKCategoryTypeIdentifierMindfulSession`
- **Method Signatures (Swift)**:
  - `@objc func requestPermission(_ call: CAPPluginCall)`
  - `@objc func logMindfulSession(_ call: CAPPluginCall)`
- **Permission Flow**:
  1. Core checks `HKHealthStore.isHealthDataAvailable()`. If false, rejects with a supportive error message.
  2. Resolves `HKObjectType.categoryType(forIdentifier: .mindfulSession)`.
  3. Invokes `healthStore.requestAuthorization(toShare: [mindfulType], read: nil)`.
- **Sample Logging**:
  - Parses ISO-8601 strings from `startDate` and `endDate`.
  - Instantiates `HKCategorySample` with value `HKCategoryValue.notApplicable.rawValue`.
  - Saves sample via `healthStore.save()`.
- **Error States**:
  - HealthKit is not available on iPad / other devices.
  - User denies access (resolves with `granted: false` or handles permission gracefully).
  - Invalid date format passed (fails validation).

### Android (Health Connect) Implementation Detail

- **Target Record Type**: `MindfulnessSessionRecord`
- **Method Signatures (Java/Kotlin)**:
  - `@PluginMethod public void requestPermission(PluginCall call)`
  - `@PluginMethod public void logMindfulSession(PluginCall call)`
- **Permission Flow**:
  1. Verifies Health Connect app is installed and active on the device.
  2. Requests write permission for `MindfulnessSessionRecord.class` using the Jetpack `HealthConnectClient`.
- **Sample Logging**:
  - Builds a `MindfulnessSessionRecord` with `startTime` and `endTime` instants, along with an optional metadata/title field if supported.
  - Writes the record via the client.
- **Error States**:
  - Health Connect API is unavailable or app not installed. Rejects call so frontend can guide the user.
  - Permission is denied by the user.

## 2. CSV Export Specification

To support web users who do not have access to native OS health platforms, we offer a clean CSV export endpoint.

- **Endpoint**: `GET /api/v1/listening-sessions/me/sessions/export`
- **Authentication**: Authoritative `x-anon-id` or claimed `user_id` session.
- **CSV Headers**:
  - `ID`: Unique identifier of the played history log entry.
  - `Session Title`: Title of the listening session config, or "Standalone Timer" if timer.
  - `Started At`: ISO-8601 start timestamp.
  - `Completed At`: ISO-8601 end timestamp.
  - `Duration (Seconds)`: Total duration listened.
  - `Type`: "listening-session" or "standalone-timer".
- **Response Headers**:
  - `Content-Type: text/csv; charset=utf-8`
  - `Content-Disposition: attachment; filename="annealmusic_history_export.csv"`

## 3. Accessibility (a11y) Audit Checklist

All surfaces introduced in v4.0–v4.5 will be swept against **WCAG 2.1 AA** criteria.

### Checklist Items

- [ ] **Contrast (Minimum) - WCAG 1.4.3**: All normal text must have a minimum contrast ratio of 4.5:1 against its background. (Affects warm-dark timers, progress timelines, disclaimers, etc.).
- [ ] **Keyboard Navigation - WCAG 2.1.1**: Every interactive control (Play/Pause, Sliders, Bell previews, toggles) must be focusable via standard Tab sequencing and triggerable via Enter/Space.
- [ ] **Focus Visible - WCAG 2.4.7**: Every focusable element must have a clear, high-visibility focus indicator (e.g. glowing border, amber outline).
- [ ] **Name, Role, Value - WCAG 4.1.2**: All controls must have descriptive accessible names (e.g. `aria-label="Play Session"`, `aria-label="Adjust settle-in duration"`).
- [ ] **Reduced Motion - WCAG 2.3.3**: Visual visualizer animations (particles, orbits) and breathing expand-contract animations must halt or decrease velocity when `prefers-reduced-motion` is active.
- [ ] **Touch Target Size**: All mobile click targets must have a minimum size of 44×44 CSS pixels (points).
- [ ] **Color Independence - WCAG 1.4.1**: Color is not used as the sole visual means of conveying information, indicating an action, prompting a response, or distinguishing a visual element.

## 4. Remediation Plan

### Visual Contrast

- Refactor tailwind styling in `ListeningView.tsx` and `MeditationTimerPage.tsx` to elevate low-contrast text (e.g. from `text-stone-500` to `text-stone-400` or `text-amber-200/90` depending on the surrounding tone).
- Elevate disclaimer footprint contrast slightly while keeping its subtle visual weight.

### Keyboard Navigation & Focus

- Inject `focus-visible:ring-2 focus-visible:ring-amber-500/80 focus:outline-none` on all standard and custom buttons/toggles.
- Add proper ARIA roles and keyboard listener support to the Timeline Bell markers.

### Screen Readers

- Add dynamic `aria-live="polite"` announcers inside the HUD that speak aloud state changes (e.g. "Now entering Deep Listening phase").
- Ensure all sliders have descriptive `aria-valuetext` readouts.

### Touch Targets

- Increase layout padding of the escape hatch triggers, timer close buttons, and accordion control buttons so they reach 44×44 points without sacrificing aesthetics.

### Reduced Motion

- Bind `window.matchMedia('(prefers-reduced-motion: reduce)')` in both `Visualizer.tsx` and `MeditationTimerPage.tsx` to lock animations to static or near-static states.

## 5. Performance Audit Plan

To close out the release with high confidence, we will track the following mobile metrics:

1. **Cold Start Latency**: Time from application boot to first synthesized sound must be `< 3s` on mid-range target devices.
2. **Thermal Stability**: Execute a continuous 30-minute session and assert no thermal throttling, audio dropouts, or worker congestion.
3. **Battery Footprint**: Profile and document the battery consumption rate (delta) comparing active listening visualizer runs to idle states.

## 8. Risks & Mitigations

### Risk: HealthKit / Health Connect API surface changes

- **Mitigation**: We wrap all native calls inside `try-catch` blocks and use evergreen standard Jetpack/HealthKit namespaces. Fallbacks return cleanly to prevent app crashes.

### Risk: Accessibility remediation exposes larger architectural overhaul

- **Mitigation**: We isolate contrast and layout changes behind CSS tokens and utilities. If complex custom components fail a11y in a way that requires complete rewrites, we document the edge cases and carry them to the immediate `v4.6.1` patch.
