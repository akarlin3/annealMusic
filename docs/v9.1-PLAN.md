# AnnealMusic v9.1 · Aesthetic System · Release Plan

This document represents the comprehensive blueprint and design system specifications for **v9.1: Aesthetic System**.

The goal is to implement a **unified design system** with **per-mode aesthetic variations**. The warm-amber-dark identity persists across all modes; what varies is density, motion, typography weights, contrast, and ornamentation.

---

## 1. Full Token System Specification

The system is defined as **one design language with three voices**. Tokens are defined in a centralized TypeScript configuration (`src/design/tokens.ts`) and applied to the DOM via dynamic CSS variables scoped by `[data-mode="meditation"]`, etc.

### Core Variables Mapping

We establish a canonical, semantic set of CSS variables that components consume:

```css
:root {
  /* Shared Base Scales */
  --font-family-display: 'Instrument Serif', Georgia, serif;
  --font-family-body: 'Geist', system-ui, sans-serif;
  --font-family-mono: 'Geist Mono', ui-monospace, monospace;

  --spacing-scale-base: 4px;

  /* Shared Color Bases */
  --color-rose: 248, 113, 113; /* rgb components for opacity manipulation */
  --color-cyan: 125, 211, 252;
}
```

## 2. Component Library Catalog & Scope

Every UI element is audited to consume the token system instead of inline CSS styles.

### In Scope for v9.1

We will construct canonical implementations of core primitives in `src/design/components/`:

1. **Button** (`src/design/components/Button.tsx`):
   - Primary: Solid amber fill, active scale micro-animations.
   - Secondary: Bordered outline, subtle fill on hover.
   - Ghost: Raw text, color shift on hover.
   - Destructive: Solid rose/red, safe for critical operations.
2. **Slider** (`src/design/components/Slider.tsx`):
   - Fully continuous sliding inputs, keyboard focus targets, active glowing thumb overlays.
   - Minimum 44×44px hit-target for accessibility.
3. **Panel** (`src/design/components/Panel.tsx`):
   - Modals, drawers, and overlay popovers.
   - Built-in dynamic backdrop filter (`backdrop-blur-md`) and responsive transitions matching active motion duration multipliers.
4. **Card** (`src/design/components/Card.tsx`):
   - Standard panels for loop slots, lesson lists, or curated listening paths.

### Deferred to v9.3

- Complex multi-stage diagnostic modals.
- Admin-specific telemetry configuration panels.
- Profile editing overlays.

## 3. Motion Language Across Modes

Visual elements utilize CSS variables mapped to the dynamic multiplier:

- **Meditation**: Long, graceful fades. Transitions default to `duration * 1.2` (e.g. 350ms → 420ms).
- **Musician**: High tactile engagement. Transitions default to `duration * 1.0` (e.g. 250ms).
- **Researcher**: Zero lag. Transitions default to `duration * 0.8` (e.g. 150ms) or instantaneous jumps.
- **Accessibility Override**: If `prefers-reduced-motion: reduce` is enabled, all transition scales are overridden to `0s`, replacing motion with absolute instantaneous value swaps.

---

## 4. Typography Rules

The typography leverages `Instrument Serif` (Display) + `Geist` (Body) + `Geist Mono` (Tabular) and scales body and heading weights dynamically:

- **Meditation**: Bodyshop uses weight 300, leading 1.75. Provides spaciousness.
- **Musician**: Body weight 400, leading 1.5. Clean, standard reading.
- **Researcher**: Body weight 450/500, leading 1.4. Tabular data figures explicitly enforce `font-variant-numeric: tabular-nums` and use `Geist Mono` for precise alignment of frequencies and measurements.

---

## 5. Icon Set Variations

We will leverage `lucide-react` icons. To avoid feeling gimmicky, we adjust parameters:

- **Meditation**: Thin stroke lines (`strokeWidth={1.2}`), softer shapes.
- **Musician**: Standard stroke lines (`strokeWidth={1.8}`), balanced creative shapes.
- **Researcher**: Thick, clear contrast lines (`strokeWidth={2.2}`), blocky/geometric shapes where possible.

For specific concepts:

- **Play**: Meditation uses the Wind/Breath symbol or a highly rounded play icon; Musician uses a standard music play icon; Researcher uses a literal solid sharp triangle.

---

## 6. Visualizer Preset Specifications

Enhance both the Canvas fallback (`drawFrame`) and WebGL renderer (`visual.frag.glsl`) to integrate mode-specific values:

### A. WebGL presets

1. **Meditation Preset**:
   - Brightness: `u_calm` scales the overall brightness by `0.45` (`color * 0.45`).
   - Bloom & Blur: Gentler, wider bloom factor (`glowRadius` scaled by `1.2x`, alpha cut by `0.5x` for soft ambient fuzziness).
   - Speed: Slow orbital motion (`speedScale` scaled by `0.45`).
   - Spectrum: Disabled/hidden by default to prevent visual distractions.
2. **Musician Preset**:
   - The standard responsive, audio-reactive orbit particles from v1.9.
3. **Researcher Preset**:
   - High information contrast.
   - Spectrum trace visible at the bottom with higher contrast (`opacity 0.6`).
   - **Partial Frequency Labels**: Absolute absolute React-based HUD overlay directly on top of the visualizer. Renders text nodes matching coordinates (`px`, `py`) of the active partials (e.g., `440.0 Hz (A4)`) dynamically!

### B. Canvas presets

The 2D canvas fallback matches these visual parameters:

- Meditation cuts draw speeds to `0.45x`, sets low-contrast gradients.
- Researcher draws text annotations directly next to each coordinate circle on every frame using `ctx2d.fillText`.

## 7. Migration & Backward-Compat Strategy

To prevent visual drift or breaking changes for existing components:

1. **Soft CSS Var Injection**: Inject CSS variables via a React context provider wrapper (`ModeAesthetic`) that targets `document.documentElement` dynamically.
2. **Inheritance Fallbacks**: Existing Tailwind styles continue to work as is; custom elements are systematically migrated component-by-component.
3. **Zero Inline Color Policy**: All custom background colors inside files (e.g. `bg-[#0c0a09]`) are replaced with `bg-[var(--bg-base)]`.

---

## 8. Accessibility Audit Plan

- **WCAG 2.1 AA Compliance**: Contrast ratios verified across all mode variations (text vs background must remain ≥ 4.5:1).
- **Reduced Motion**: Verify CSS media query `prefers-reduced-motion` cancels all motion variables.
- **Keyboard Navigation**: Ensure custom sliders are accessible via left/right arrows, and buttons can be selected with Enter/Space.
- **Target Size**: Minimum touch targets are locked to 44px by wrapping smaller buttons in invisible padding frames.

---

## 9. Key Risks & Mitigation

1. **Visual Drift**: Users might feel disoriented by the subtle differences.
   - _Mitigation_: Per-mode variations are kept subtle, elegant, and highly polished, maintaining the core warm amber theme.
2. **WebGL Context Interruption**: Mode switching could interrupt the GPU context.
   - _Mitigation_: The theme swap is completely driven by CSS variables on the document element, so the WebGL canvas element is NEVER re-mounted, preserving the rendering context flawlessly.
