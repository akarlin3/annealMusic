# AnnealMusic v3.3 · Meta-Arcs · Technical Specification & Plan

This document details the architectural design, mathematical models, collaborative networking semantics, offline rendering guidelines, and user interface features to introduce procedural **meta-arc** segments to AnnealMusic timeline pieces.

---

## 1. Meta-Arc Data Model & URL Schema Version 11 Spec

### 1.1 Data Model

In `src/piece/types.ts`, the `SegmentType` will be extended to support `meta-arc`:

```typescript
export type SegmentType = 'fixed' | 'arc' | 'open' | 'transition' | 'meta-arc';
```

A `meta-arc` segment carries kind-specific config parameters and a nullable `seed` in its `config` object:

```typescript
export interface MetaArcConfig {
  kind:
    | 'random-walk'
    | 'waypoint-tour'
    | 'bell-curve-variation'
    | 'spectral-evolution';
  seed: string | number | null; // null represents dynamic evaluation per play

  // Kind-specific configuration parameters:
  randomWalk?: {
    params: ArcTargetKey[];
    driftStrength: number; // theta-like / sigma noise, e.g. 0.05..0.5
    meanReversion: number; // mean reversion pull, e.g. 0.05..0.5
    steps: number; // number of linear segments, e.g. 10..50
    bounds: Partial<Record<ArcTargetKey, { min: number; max: number }>>;
  };
  waypointTour?: {
    params: ArcTargetKey[];
    waypointsCount: number; // e.g. 3..10
    maxDistance: number; // e.g. 0.1..1.0
    easing: CurveName;
    bounds: Partial<Record<ArcTargetKey, { min: number; max: number }>>;
  };
  bellCurveVariation?: {
    params: ArcTargetKey[];
    paramBounds: Partial<Record<ArcTargetKey, { min: number; max: number }>>;
    minSettleFraction: number; // e.g. 0.1..0.4
    maxSettleFraction: number; // e.g. 0.2..0.5
    minHoldFraction: number; // e.g. 0.1..0.4
    maxHoldFraction: number; // e.g. 0.2..0.5
  };
  spectralEvolution?: {
    rootBounds: { min: number; max: number };
    densityBounds: { min: number; max: number };
    brightnessBounds: { min: number; max: number };
    coordinationType: 'inverse-crossover' | 'parallel-sweep';
  };
}
```

### 1.2 URL Schema Bumps to Version 11

We bump `SCHEMA_VERSION` in `src/share/schema.ts` to `11`.

- When encoding a piece with a `meta-arc` segment, we serialize the segment properties using flat, namespaced query parameter indices under `seg<idx>` to avoid URL length bloop:
  - `seg<idx>.type=meta-arc`
  - `seg<idx>.dur=<duration>`
  - `seg<idx>.kind=<kind>`
  - `seg<idx>.seed=<seed>` (or `null`)
- Based on `kind`, we encode the specific config sub-fields, e.g.:
  - `seg0.walk.params=rootFreq,brightness&seg0.walk.drift=0.15&seg0.walk.mean=0.1&seg0.walk.steps=25`
  - `seg0.walk.bounds=rootFreq:0.5,1.5;brightness:0.3,0.9`

**Backward Compatibility Policy**:

- Pieces of version <= 10 continue to parse flawlessly. Default fallback maps `meta-arc` segments to a standard `fixed` segment if loaded on older builds.

## 2. Generator Interface

Each of the 4 meta-arc generators is a pure function of `(config, seed)` which produces a standard `Arc` object. No external clocks, system dates, or Math.random can be utilized inside the generator logic.

```typescript
export type MetaArcGenerator = (
  config: Record<string, any>,
  seed: string | number,
) => Arc;
```

### Seeded PRNG Scaffolding

We use a robust sfc32 or mulberry32 pseudo-random number generator for uniform randomness:

```typescript
export function createSeededRng(seed: string | number): () => number {
  let s = typeof seed === 'number' ? seed : hashStringToInt(seed);
  s = s >>> 0;
  return () => {
    s = (s * 1664525 + 1013904223) >>> 0;
    return s / 0xffffffff;
  };
}

function hashStringToInt(str: string): number {
  let h = 5381;
  for (let i = 0; i < str.length; i++) {
    h = (h << 5) + h + str.charCodeAt(i);
  }
  return h >>> 0;
}
```

## 3. Per-Kind Generator Specs

### 3.1 Kind 1: `random-walk`

- **Concept**: Ornstein-Uhlenbeck-like discrete walk on a configurable set of sculpt params, within custom bounds.
- **Math**:
  At each step $i$ from $1 \dots N$ (where $N = \text{steps}$):
  $$X_i = X_{i-1} - \theta (X_{i-1} - M) \Delta t + \sigma (R_i - 0.5) \sqrt{\Delta t}$$
  where:
  - $\theta = \text{meanReversion}$ (pull strength towards the baseline, default 0.1)
  - $\sigma = \text{driftStrength}$ (noise amplitude, default 0.1)
  - $M = 1.0$ (neutral baseline parameter multiplier)
  - $\Delta t = 1.0 / N$
  - $R_i \sim \text{Uniform}(0, 1)$ via seeded RNG
  - $X_0 = 1.0$
- **Clamp**: Each step state $X_i$ is clamped inside the custom bounds `[min, max]` for that parameter.
- **Output**: Generates $N$ contiguous `ArcSegment`s with fraction $1/N$, linear easing, and the calculated parameter multipliers as target values.

### 3.2 Kind 2: `waypoint-tour`

- **Concept**: Generator picks $N$ random "waypoints" (param configurations) within user-defined bounds and visits them sequentially.
- **Algorithm**:
  - Generate waypoints $W_0, W_1, \dots, W_{N-1}$.
  - $W_0$ starts as the neutral multiplier ($1.0$ for each param).
  - For each subsequent waypoint $W_i$ ($i \ge 1$):
    - Select each parameter multiplier $W_i[p]$ randomly within `bounds[p]`.
    - Apply a maximum distance delta filter relative to the previous waypoint:
      $$|W_i[p] - W_{i-1}[p]| \le \text{maxDistance} \times (\text{bounds}[p].\text{max} - \text{bounds}[p].\text{min})$$
  - **Output**: Generates $N - 1$ contiguous `ArcSegment`s with fraction $1 / (N - 1)$, using the user-defined `easing` curve, each targeting the parameter state of waypoint $W_i$.

### 3.3 Kind 3: `bell-curve-variation`

- **Concept**: A randomized variation of the signature "Open, Deepen, Return" structural arc.
- **Algorithm**:
  - Settle fraction $f_1 \sim \text{Uniform}(\text{minSettle}, \text{maxSettle})$.
  - Hold fraction $f_2 \sim \text{Uniform}(\text{minHold}, \text{maxHold})$.
  - Return fraction $f_3 = 1.0 - f_1 - f_2$. (If sum exceeds bounds, fractions are normalized to sum to $1.0$).
  - For each parameter $p$:
    - Settle target $T_{\text{settle}} \sim \text{Uniform}(\text{bounds}[p].\text{min}, \text{bounds}[p].\text{max})$.
    - Hold target $T_{\text{hold}}$ is picked within a small variance ($\pm 10\%$) of $T_{\text{settle}}$, clamped to bounds.
  - **Output**: Generates 3 segments:
    - Segment 1: fraction $f_1$, easing `easeInOut`, targets $T_{\text{settle}}$.
    - Segment 2: fraction $f_2$, easing `linear`, targets $T_{\text{hold}}$.
    - Segment 3: fraction $f_3$, easing `easeInOut`, targets `'restoreStart'`.

### 3.4 Kind 4: `spectral-evolution`

- **Concept**: Coordinate the temporal progression of `rootFreq`, `density`, and `brightness`.
- **Algorithm**:
  - Generate 3 segments with random fractions $f_1, f_2, f_3$ summing to $1.0$ (e.g. baseline $[0.33, 0.34, 0.33]$ with small seed-based jitter).
  - **Inverse Crossover**:
    - Waypoint 1 ($W_1$): `density` goes high (e.g. $80..100\%$ of bounds), `brightness` goes low (e.g. $0..20\%$ of bounds), `rootFreq` drops (lower half of bounds).
    - Waypoint 2 ($W_2$): crossover! `density` goes low ($0..20\%$), `brightness` goes high ($80..100\%$), `rootFreq` rises (upper half of bounds).
    - Waypoint 3 ($W_3$): settle towards baseline averages (neutral $1.0$ or midpoint of bounds).
  - **Parallel Sweep**:
    - $W_1$: `density` and `brightness` both sweep high, `rootFreq` sweeps low.
    - $W_2$: `density` and `brightness` both sweep low, `rootFreq` sweeps high.
    - $W_3$: settle to midpoints.
  - **Output**: 3 segments targeting $W_1, W_2, W_3$ using `easeInOut` curves.

## 4. UI: Per-Kind Config Panels + Preview Generation

### 4.1 Properties Drawer Config UI

In `src/piece/SegmentProperties.tsx` when a segment of type `meta-arc` is active:

- **Tabs**: Render a responsive tab group for kind selection (`Walk`, `Tour`, `Bell`, `Spectral`).
- **Sliders Drawer**: Render parameters associated with the chosen generator (e.g. drift strength, steps count, waypoint counts).
- **Padlock Lock-Seed UX**:
  - Unlocked: seed is `null` (text label: "Dynamic Generation").
  - Click to Lock: rolls a random seed, saves it to config, locking the padlock.
  - Re-roll button: rolls a new seed without unlocking.

### 4.2 Canvas Waveform Preview

- An inline `<canvas>` preview rendering the curves in real time.
- Clicking "Preview generation" (or making slider changes) executes the generator, creates an `ArcRunner`, and steps through 100 intervals.
- The parameter sweeps are plotted as smooth, colorful glowing paths (e.g. Cyan for Root, Purple for Brightness, Orange for Space) on a dark glassmorphic grid background.

## 5. Collab Semantics for Null Seeds

To solve audio divergence when a meta-arc has a `null` seed in a shared Jam room:

1. **Leader Role**: The session leader (creator of the Jam room) rolls a random seed, and writes it to the shared Yjs `sessionConfigMap` under `meta_arc_seed_${segmentIdx}`.
2. **Follower Role**: Follower clients observe updates to `sessionConfigMap` and retrieve the seed to generate the identical arc.
3. **Safety Fallback**: If a follower enters the segment and the leader's seed is not received within $250\text{ms}$, the follower rolls a local random seed to maintain audio playback continuity.

---

## 6. Stem Export Determinism Integration

Offline stem rendering must remain bit-identical across runs.

- If a `'meta-arc'` segment has a `null` seed, we derive a deterministic seed from the global export `config.seed` and the `segmentIdx`:
  $$\text{derivedSeed} = \text{config.seed} + \text{segmentIdx}$$
- Re-rendering a piece with the same seeds produces bit-identical wave files.

## 7. Risks & Mitigation Plan

| Risk                                                 | Mitigation                                                                                                                                                       |
| :--------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Generators producing values outside safe limits.** | Generators pass all computed states through the robust `clampParam` utility before packaging them into the `Arc` segments, ensuring audio safety.                |
| **URL length limits in v11.**                        | Keep config structures tightly compressed, serializing only parameters that deviate from the defaults, keeping characters well below 1,000.                      |
| **Collaborative lag on leader seed broadcast.**      | Followers use a short fallback timeout ($250\text{ms}$) where they begin playing with a local seed rather than freezing or crashing if the leader drops offline. |
