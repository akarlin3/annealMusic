## 1. Analytics: schema + materialized view

`0024_v6_5_lesson_analytics` creates the brief's rollup as a Postgres `MATERIALIZED VIEW`
(guarded `if is_pg:`; on SQLite it is skipped ‚Äî harmless, since migrations don't run in
tests and SQLite is dev/test only). Because `abandoned` is **never stored**, the view does
not filter on it; abandonment is computed in the service from `last_active_at`.

```sql
CREATE MATERIALIZED VIEW lesson_analytics AS
SELECT
  lesson_id,
  COUNT(*)                                              AS views,
  COUNT(*) FILTER (WHERE state = 'completed')           AS completions,
  COALESCE(AVG(EXTRACT(EPOCH FROM (completed_at - started_at)) * 1000)
           FILTER (WHERE state = 'completed'), 0)::INTEGER AS avg_completion_ms,
  COUNT(reflection_text) FILTER
    (WHERE reflection_text IS NOT NULL AND reflection_text <> '') AS reflections
FROM lesson_progress
GROUP BY lesson_id;
CREATE UNIQUE INDEX idx_lesson_analytics_lesson ON lesson_analytics(lesson_id);
```

**Refresh:** `REFRESH MATERIALIZED VIEW CONCURRENTLY lesson_analytics` (needs the unique
index) ‚Äî nightly (ops/cron, documented) + on-demand via the admin endpoint. Postgres-only;
the refresh endpoint is a documented no-op on SQLite.

**Tested code path:** to avoid an untested Postgres-only branch, the admin endpoints
**compute aggregates live** with portable SQLAlchemy Core `GROUP BY` queries over the base
tables (bounded by `lesson_progress` volume). The materialized view is the **production
performance / external-BI** artifact (refreshed nightly + on demand); it is intentionally
not the only source. This is the honest tradeoff for a SQLite-tested harness with a
Postgres-only view (recorded in the CP1 report).

## 2. Per-step drop-off + on-demand metrics (`app/services/analytics.py`)

All computed from `step_actions` joined to `lesson_steps` (for step `type` and, for
audio-clip steps, `config.clip_slug`). Pure helper functions take plain dicts so they unit-
test without a DB:

- **Drop-off curve:** total steps = `count(lesson_steps for lesson)`. For each step index
  `i`, `reached[i] = #progress rows whose max viewed step_position ‚â• i`. Curve =
  `reached[i]/reached[0]`. The sharp drop = the abandon cliff.
- **Per-step time-on-step:** aggregate `ms` per `step_position` from `viewed` actions ‚Üí
  mean / median / p90 (discard absurd gaps as walk-aways; calm-by-design ‚Äî pausing is fine).
- **Audio-clip play counts:** per `clip_slug`, count clip actions on audio-clip steps. The
  v6.3 data only has `viewed`, so a `viewed` on an audio-clip step is the **exposure** signal
  today; **CP3 additively wires** `onClipAction ‚Üí 'clip_play' / 'clip_replay'` so true play /
  replay counts begin to flow. Skip rate = audio-clip-step exposures with no play.
- **Prompt "I tried it" vs skip:** CP3 additively emits `'prompt_tried' / 'prompt_skipped'`
  from `PromptStep`; the service counts them per step. Until then the metric is honestly 0.
- **Reflection presence rate:** `reflections / views` (from the rollup).

These new action strings are **additive** (the upsert already accepts arbitrary
`step_actions` dicts) ‚Äî no schema/API break.

## 3. Admin endpoints (`app/routers/analytics_admin.py`)

`APIRouter(prefix="/api/v1/admin/analytics", dependencies=[Depends(require_admin)])`:

| Method ¬∑ Path | Returns |
|---|---|
| `GET /lessons` | per-lesson rollup list (views, completions, completion_rate, avg_completion_ms, abandonments, reflections, reflection_rate) |
| `GET /lessons/{lesson_id}` | rollup + step drop-off curve + per-step time dist + prompt tried/skip + clip stats |
| `GET /tracks` | per-track aggregate completion + **path popularity** (observed adjacent lesson transitions vs the prerequisite graph) |
| `GET /clips` | per-clip play / replay / skip / exposure counts |
| `POST /refresh` | refresh the materialized view (PG); `{refreshed_at}` (no-op note on SQLite) |

- **Path popularity:** order each user's rows by `started_at` within a track; tally adjacent
  `(from‚Üíto)` transitions; compare to `Lesson.prerequisites`. Returns top observed paths +
  "off-graph" transitions. Aggregate counts only.
- Response schemas in `app/schemas.py` (or a small `schemas` addition) ‚Äî plain `BaseModel`.
- **Privacy:** every query `GROUP BY`/`COUNT`s before returning; **no `user_id`/PII** ever in
  a payload. A test asserts no analytics response contains a user identifier.

## 4. Discoverability surfaces ‚Äî touchpoints + format

One primitive (`src/components/LessonHintLink.tsx`) keeps the heuristic in one place:

```tsx
<LessonHintLink lessonPath="synthesis-fundamentals/fm-engine"
  label="Learn more about this engine" variant="link" />
```
- Opens `/learn#/lesson/<path>` in a **new tab** (`target="_blank" rel="noopener"`).
- `variant="link"` (subtle inline link) or `"icon"` (understated `?` with accessible label).
- Renders nothing when learning hints are hidden (host passes the global flag).

Touchpoints (all gated on the global hide toggle, understated by default):

|---|---|---|
| `components/EngineSelector.tsx` | "Learn more about this engine" link | engine‚Üílesson map |

| `components/ModeToggle.tsx` | per-mode "Learn the ‚Ä¶ track" | mode‚Üítrack map |

## 5. Hide-learning-hints toggle

- A global preference, default **on** (opt-out), surfaced as a checkbox in
  `src/pages/AccountSettingsPage.tsx` ("Show learning hints").
- Persisted in the existing client preference store (or `localStorage` with a small
  `useLearningHints()` hook if no store field exists) ‚Äî **verified at implementation time**;
  reuse the established pattern rather than inventing one.
- When off: every `LessonHintLink` host renders nothing and the banner is suppressed.
- Calm-by-design: hints are quiet, never animated, never nagging; the toggle is the opt-out.

## 8. Risks

| Risk | Mitigation |
|---|---|
| Materialized view is Postgres-only; tests are SQLite (no migrations) | endpoints compute live via portable `GROUP BY`; view is the prod perf/BI artifact; refresh endpoint PG-guarded |
| Brief metrics (clip play/replay, prompt tried/skip) not in v6.3 data | service reads them when present; CP3 additively emits the new action strings; metrics degrade to honest 0 until data accrues |
| Query perf at scale | materialized view for the hot rollup; per-lesson detail bounded to one lesson's rows |
| Exposing per-user data | aggregate-before-return; explicit no-PII test |
| Discoverability noise / engagement creep | one understated primitive; global opt-out; no counts/streaks/nags; calm CI lexical gate covers new copy |
| Heuristic drift | single `LessonHintLink` + single `lessonHints.ts` |
