## 1. Encoding format (chosen) + worked example

**Format:** `#s=<version>:<payload>` where the payload is **readable
`key=value` pairs joined by `&`**, keys are the full param names, values are
fixed-decimal numbers.

Rationale: with 7 numeric params the URL is ~90 chars ‚Äî far under the 200/500
thresholds ‚Äî so we keep the fully human-decodable form and skip base64. Full key
names (not 1-char abbreviations) maximize debuggability; the few extra bytes are
free at this length.

**Payload grammar:**

```
payload := pair ("&" pair)*
pair    := key "=" number
key     := one of SHARED_KEYS
number  := value formatted to that key's decimal precision
```

**Worked end-to-end example** (a fully-sculpted session):

```
https://anneal.averykarlin.org/#s=1:rootFreq=147&spread=1.08&density=7&coupling=0.62&drift=0.30&brightness=0.74&space=0.55
```

Default-state URL (every param at its default):

```
https://anneal.averykarlin.org/#s=1:rootFreq=110&spread=1.00&density=6&coupling=0.30&drift=0.50&brightness=0.50&space=0.40
```

Length of the fragment payload above: ~95 chars including `s=1:`. Comfortable.

All payload characters (`a‚Äìz`, `0‚Äì9`, `=`, `&`, `.`) are valid in a URL fragment
and are not percent-encoded by browsers, so copy ‚Üí paste round-trips literally.

## 2. Schema definition

### Naming decision (flagged tradeoff)

The prompt says "rename `PhasefieldParams` ‚Üí `AnnealParams` if not already done."
The type is already `AnnealMusicParams`. **Recommendation: keep
`AnnealMusicParams`** rather than rename to `AnnealParams` ‚Äî renaming would churn
the audio engine, hooks, components, and tests for zero behavioral gain. The
share modules will import and use `AnnealMusicParams`. (If you'd prefer the exact
`AnnealParams` name, I can add a one-line `export type AnnealParams = AnnealMusicParams`
alias ‚Äî say so before CP1.)

### Included keys (`SHARED_KEYS`)

Exactly the `CONTROL_DEFS` keys (volume excluded by design):

| key          | bounds (min‚Äìmax) | decimals | example |
| ------------ | ---------------- | -------- | ------- |
| `rootFreq`   | 55 ‚Äì 220         | 0        | `147`   |
| `spread`     | 0.7 ‚Äì 1.3        | 2        | `1.08`  |
| `density`    | 2 ‚Äì 8            | 0        | `7`     |
| `coupling`   | 0 ‚Äì 1            | 2        | `0.62`  |
| `drift`      | 0 ‚Äì 1            | 2        | `0.30`  |
| `brightness` | 0 ‚Äì 1            | 2        | `0.74`  |
| `space`      | 0 ‚Äì 1            | 2        | `0.55`  |

`volume` is **never** encoded or decoded.

### Precision (decimals)

Derived from each control's `step`: `step >= 1` ‚Üí 0 decimals, else 2 decimals.
This matches the slider granularity, so encode‚Üídecode is exact (round-trip is
lossless at slider resolution).

### Bounds re-validation on decode

Every decoded value is passed through the existing `clampParam(key, value)` and
rounded to the key's decimals. Out-of-range ‚Üí clamped (+ warning). NaN ‚Üí dropped
(+ warning).

### `KEY_BOUNDS` (drift-safe)

`schema.ts` builds `KEY_BOUNDS` by reading `min`/`max`/`step`(‚Üí`decimals`) from
`CONTROL_DEFS` filtered to `SHARED_KEYS`. **No literal bounds are written in
`schema.ts`.** A test asserts `KEY_BOUNDS` agrees with `CONTROL_DEFS` for every
shared key (guards future manual edits).

## 3. Module plan (`src/share/`)

### `src/share/schema.ts`

```ts
export const SCHEMA_VERSION = 1;
export const SHARED_KEYS: readonly ParamKey[]; // ['rootFreq', ... 'space']
export interface KeyBound {
  min: number;
  max: number;
  decimals: number;
}
export const KEY_BOUNDS: Record<SharedKey, KeyBound>; // derived from CONTROL_DEFS
export function decimalsForStep(step: number): number; // step>=1 ? 0 : 2
```

### `src/share/encode.ts`

```ts
export function encodeParams(params: AnnealMusicParams): string;
//   ‚Üí 'rootFreq=147&spread=1.08&...' (payload only, no '#s=1:' prefix)
export function decodeParams(payload: string): {
  params: Partial<AnnealMusicParams>;
  warnings: string[];
};
//   ‚Üí never throws; clamps, drops unknowns/NaN, collects warnings
```

### `src/share/url.ts`

```ts
export function readStateFromHash(): {
  params: Partial<AnnealMusicParams>;
  warnings: string[];
} | null;
//   parses location.hash; null when no 's=' state or unknown version
export function writeStateToHash(params: AnnealMusicParams): void; // history.replaceState
export function subscribeStoreToHash(store, debounceMs = 500): () => void; // returns unsubscribe
```

`readStateFromHash` parsing steps: strip leading `#` ‚Üí require `s=` prefix ‚Üí
split on **first** `:` ‚Üí parse version int ‚Üí if `!== SCHEMA_VERSION` return
`null` ‚Üí `decodeParams(rest)`.

`writeStateToHash` builds `${SCHEMA_VERSION}` + payload and calls
`history.replaceState(history.state, '', '#s=1:' + payload)` so the fragment
updates without a history entry and without a navigation/scroll jump.

## 4. State store integration

Two integration points, both already located:

**(a) `src/state/params.ts` ‚Äî add a batch setter** (single state update, no
`setParam` loop):

```ts
setMany: (partial: Partial<AnnealMusicParams>) =>
  set((state) => {
    const next = { ...state.params };
    for (const k of Object.keys(partial) as ParamKey[]) {
      const v = partial[k];
      if (v !== undefined) next[k] = clampParam(k, v);
    }
    return { params: next };
  });
```

(Clamp again here as defense-in-depth; decode already clamps.)

**(b) `src/pages/App.tsx` ‚Äî boot hydration + live sync.** A small effect on
mount:

1. `const hydrated = readStateFromHash();`
2. if non-null ‚Üí `useParamStore.getState().setMany(hydrated.params)`; if
   `hydrated.warnings.length` or any keys loaded ‚Üí show toast "Loaded shared
   session" (warnings ‚Üí `console.debug`).
3. `const unsub = subscribeStoreToHash(useParamStore);` ‚Üí return `unsub` from the
   effect for unmount cleanup.

Order guarantees: hydration `setMany` runs before we subscribe, and the engine is
only built on the later Begin click ‚Äî so hydration always precedes engine
construction. StrictMode double-invoke is safe (setMany is idempotent for the
same payload; toast is single-slot). A `useRef` guard prevents a duplicate toast
under StrictMode.

  same pill visual language (`rounded-full`, `1px solid #44403c`, warm-amber

  - Failure / unsupported ‚Üí `document.execCommand('copy')` fallback using a

## 6. Failure modes (exact behavior)

| Condition                                           | Behavior                                                                                                                                           |
| --------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| No hash / no `s=` prefix                            | `readStateFromHash` ‚Üí `null`; app uses `DEFAULT_PARAMS`; no toast                                                                                  |
| Malformed payload (e.g. `s=1:garbage`, missing `=`) | decode drops unparseable pairs (+warnings); valid pairs still applied; defaults for the rest; toast "Loaded shared session" only if ‚â•1 key applied |
| Out-of-range value (`coupling=9`)                   | clamped to bound (+warning); applied                                                                                                               |
| NaN / non-numeric value (`drift=abc`)               | dropped (+warning); key keeps default                                                                                                              |
| Unknown key (`tempo=5`)                             | ignored (+warning) ‚Äî forward-compat                                                                                                                |
| Unknown schema version (`s=0:`, `s=999:`)           | `readStateFromHash` ‚Üí `null`; defaults load; toast "Link from a newer version ‚Äî can't load it"                                                     |
| Clipboard denied / unsupported                      | `execCommand` fallback ‚Üí else manual-copy toast/modal; never throws                                                                                |

`decodeParams` and `readStateFromHash` **never throw** ‚Äî guaranteed by the fuzz
test in ¬ß7.

## 7. Test plan (`src/share/__tests__/`)

1. **Round-trip (property-based, hand-rolled seeded RNG ‚Äî no new dep):** for N
   random in-bounds param sets, `decodeParams(encodeParams(p)).params` equals `p`
   for every shared key within precision. (Seeded so failures reproduce.)
2. **Fuzz decode:** 1000 random strings (incl. empty, huge, unicode, `&`/`=`
   soup) ‚Üí `decodeParams`/`readStateFromHash` never throw; always return a
   well-formed object or `null`.
3. **Out-of-bounds clamping:** values past `KEY_BOUNDS` are clamped and a warning
   is emitted.
4. **Unknown keys ignored** with a warning; known keys in the same payload still
   apply.
5. **Schema version mismatch:** `s=0:` and `s=999:` both ‚Üí `readStateFromHash`
   returns `null`.
6. **Drift guard:** `KEY_BOUNDS` agrees with `CONTROL_DEFS` (min/max/decimals)
   for every shared key.
7. **Store `setMany`:** batch update clamps and produces a single new `params`
   object.
8. **Copy-link component test (CP2):** clipboard mocked; click writes expected
   URL and flips label to `Copied`; denied-clipboard path exercises fallback.

`history.replaceState` / `location.hash` are exercised under jsdom (both
supported).

## 8. Tradeoffs flagged

1. **Readable `key=value&‚Ä¶` over base64.** Chosen for debuggability; only valid
   because length stays tiny. If a future schema balloons past ~500 chars (e.g.
   arc timelines in v0.4), revisit with base64-of-JSON ‚Äî noted for later.
2. **Keep `AnnealMusicParams`, don't rename to `AnnealParams`.** Avoids
   cross-codebase churn; offered an alias if the exact name is wanted (¬ß2).
3. **Mount-effect hydration vs. module-init hydration.** Spec calls for a mount
   effect + `setMany`; this cleanly carries warnings to the toast and is
   StrictMode-safe. Alternative (seed the store at module load) avoids a
   theoretical default‚Üíhydrated flash, but the engine isn't built until Begin and
   the control sliders re-render instantly, so no audible/visible flash in
   practice. Going with the spec'd approach.
4. **`&` as the pair separator** inside a fragment (vs `,`). `&` reads like a
   familiar query string and is fragment-safe; we parse it ourselves (not via
   `URLSearchParams`, to keep behavior explicit and dependency-free).
5. **Clamp twice** (decode + `setMany`). Cheap defense-in-depth; keeps the store
   authoritative even if a future caller bypasses decode.
6. **No new runtime deps.** Property testing is hand-rolled seeded RNG rather
   than adding `fast-check`, keeping the dep tree unchanged. (Can switch to
   `fast-check` as a dev-dep if you prefer richer shrinking ‚Äî flag before CP1.)
