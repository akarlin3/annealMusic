## 3. Detailed Measurement & Validation Protocol

Optimizations will be measured under standardised local and simulated environments:

1. **Bundler Analysis**: Using `rollup-plugin-visualizer` (via `npm run build` and `npm run build:research`).
2. **Cold Start TTI**: Chrome DevTools Performance panel under Simulated **Fast 3G** network throttling and **4x CPU slowdown**.
3. **Audio Thread Performance**: Emitted telemetry marks or real-time measurements in AudioWorklet processors using `performance.now()`.
4. **Memory Footprint**: Active profile inspections using Chrome Memory Profiler over continuous 10-minute and 30-minute stress tests.
5. **GATT BLE Simulation**: Emulated high-frequency GATT callbacks monitored to track state execution overhead.

---

## 4. Risk Register

- **Risk 1: DSP Output Disruption (Lagrange / Biquad optimization)**:
  - _Symptom_: Clicks, pops, or unstable sound output in Physical modeling.
  - _Mitigation_: Verify audio output immediately using physical synthesis test suites (`npm run test`).
- **Risk 2: Code-splitting and Async Loading Races**:
  - _Symptom_: User hits `/gallery` or click triggers sound before ToneJS finishes lazy-loading, leading to crashes.
  - _Mitigation_: Wrap all async dependencies behind cohesive loader boundaries and check for initialized engines before routing calls.
- **Risk 3: Polar GATT Throttling HRV loss**:
  - _Symptom_: Downsampling heart rate streams too aggressively loses fine-grained R-R interval changes (crucial for HRV calculation).
  - _Mitigation_: Ensure throttling only caps the Zustand/state updates frequency and does _not_ throw away intermediate R-R data points from the packet.

---

## 5. Order of Operations

1. **Step 1 (Checkpoint 1: Cold Start)**:
   - Execute Route-based Code Splitting in `src/main.tsx`.
   - Preload critical fonts and defer non-critical CSS/assets.
   - Verify bundle size assertions via visualizer plugins.
2. **Step 2 (Checkpoint 2: Runtime CPU)**:
   - Perform AudioWorklet parallel typed array optimization (`modal-bank.ts`).
   - Optimize waveguide/string loop indexing by removing hot-path `%` (modulos).
   - Rework `DataLogger.ts` size checks to eliminate dynamic stringifying.
   - Fix Yjs collab eager-loading.
3. **Step 3 (Checkpoint 3: Mobile & Release)**:
   - Cap Loop Buffer allocations inside `loadBuffer` and `commit`.
   - Throttle Polar GATT BLE stream emissions.
   - Lock in CI performance regression gates.
