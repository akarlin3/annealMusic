# AnnealMusic v5.1 Plan: Bidirectional Open Sound Control (OSC)

This document specifies the technical design, OSC address scheme, WebSocket wire protocol, standalone bridge helper architecture, Capacitor mobile plugins, configuration panel, rate-limiting filters, and threat model for the **v5.1 OSC Support** milestone.

---

## 1. Final OSC Address Scheme Specification

AnnealMusic’s OSC namespace directly mirrors the creative sculpt parameters, active synthesizer engine parameters, and session/spectral status. Address stability is guaranteed to be additive-only.

### State Broadcasts (Read Endpoints)

These are broadcast by AnnealMusic to external clients (e.g. SuperCollider, Max/MSP) via UDP or WebSocket.

| OSC Address                | Type Tag  | Description                                                                |
| :------------------------- | :-------- | :------------------------------------------------------------------------- |
| `/anneal/state/root`       | `f`       | Current root frequency in Hz (20.0 - 4200.0)                               |
| `/anneal/state/spread`     | `f`       | Tuning spread coefficient (0.7 - 1.3)                                      |
| `/anneal/state/density`    | `i`       | Active partial count (1 - 8)                                               |
| `/anneal/state/coupling`   | `f`       | Oscillator coupling factor (0.0 - 1.0)                                     |
| `/anneal/state/drift`      | `f`       | Phase drift coefficient (0.0 - 1.0)                                        |
| `/anneal/state/brightness` | `f`       | Spectral brightness (0.0 - 1.0)                                            |
| `/anneal/state/space`      | `f`       | Reverb decay/space parameter (0.0 - 1.0)                                   |
| `/anneal/state/volume`     | `f`       | Master output volume (0.0 - 0.8)                                           |
| `/anneal/state/engine`     | `s`       | Active engine ID (`"sine" \| "fm" \| "granular" \| "physical" \| "pulse"`) |
| `/anneal/state/mode`       | `s`       | Creative mode (`"sketch" \| "drone"`)                                      |
| `/anneal/spectrum`         | `b`       | FFT Spectrum blob: 256 bytes of raw uint8 amplitude values                 |
| `/anneal/partials`         | `ffff...` | Interleaved frequency (Hz) and amplitude per partial (density \* 2 floats) |
| `/anneal/session/state`    | `s`       | Creative session state (`"idle" \| "playing" \| "paused" \| "ended"`)      |
| `/anneal/session/elapsed`  | `f`       | Elapsed time since the session started, in seconds                         |

#### Engine-Specific State Broadcasts

Exposed dynamically based on the active engine selection under `/anneal/state/engine_params/<engine>/<param>`:

- **FM (`fm`)**:
  - `/anneal/state/engine_params/fm/modRatio` (`f`): Modulator frequency ratio (0.5 - 4.0)
  - `/anneal/state/engine_params/fm/modIndex` (`f`): Modulation depth index (0.0 - 10.0)
  - `/anneal/state/engine_params/fm/feedback` (`f`): Modulator self-feedback (0.0 - 1.0)
- **Granular (`granular`)**:
  - `/anneal/state/engine_params/granular/source` (`f`): Audio source index (0.0 - 3.0)
  - `/anneal/state/engine_params/granular/size` (`f`): Grain size in milliseconds (30.0 - 300.0)
  - `/anneal/state/engine_params/granular/density` (`f`): Grain trigger rate per second (4.0 - 40.0)
  - `/anneal/state/engine_params/granular/posJitter` (`f`): Grain position jitter (0.0 - 1.0)
  - `/anneal/state/engine_params/granular/pitchJitter` (`f`): Grain pitch jitter in cents (0.0 - 100.0)
  - `/anneal/state/engine_params/granular/posCenter` (`f`): Grain playhead center (0.0 - 1.0)
- **Physical (`physical`)**:
  - `/anneal/state/engine_params/physical/model` (`f`): Physical model index (0.0 - 7.0)
  - `/anneal/state/engine_params/physical/excitationLevel` (`f`): Continuous excite energy (0.0 - 1.0)
  - `/anneal/state/engine_params/physical/damping` (`f`): High-frequency damping factor (0.0 - 1.0)
  - `/anneal/state/engine_params/physical/brightness` (`f`): Resonator excitation brightness (0.0 - 1.0)
  - `/anneal/state/engine_params/physical/reed` (`f`): Non-linear excitation feedback stiffness (0.0 - 1.0)
  - `/anneal/state/engine_params/physical/inharm` (`f`): Modal inharmonicity factor (0.0 - 1.0)
- **Pulse (`pulse`)**:
  - `/anneal/state/engine_params/pulse/density` (`f`): Pulse grid subdivision index (0.0 - 5.0)
  - `/anneal/state/engine_params/pulse/accent` (`f`): Accent emphasis state (0.0 or 1.0)
  - `/anneal/state/engine_params/pulse/tone` (`f`): Synthesis lowpass filter tone (0.0 - 1.0)
  - `/anneal/state/engine_params/pulse/swing` (`f`): Grid shuffle swing factor (0.0 - 1.0)
  - `/anneal/state/engine_params/pulse/humanize` (`f`): Trigger timing jitter (0.0 - 1.0)

### Control Actions (Write Endpoints)

These are received by AnnealMusic to sculpt parameter state from external controllers.

| OSC Address                     | Type Tag | Action / Implemented Mapping                                              |
| :------------------------------ | :------- | :------------------------------------------------------------------------ |
| `/anneal/control/root`          | `f`      | Sets root frequency (Hz). Maps to `setParam('rootFreq', value)`           |
| `/anneal/control/spread`        | `f`      | Sets spread. Maps to `setParam('spread', value)`                          |
| `/anneal/control/density`       | `i`      | Sets active partial density (1 - 8). Maps to `setParam('density', value)` |
| `/anneal/control/coupling`      | `f`      | Sets coupling. Maps to `setParam('coupling', value)`                      |
| `/anneal/control/drift`         | `f`      | Sets drift. Maps to `setParam('drift', value)`                            |
| `/anneal/control/brightness`    | `f`      | Sets brightness. Maps to `setParam('brightness', value)`                  |
| `/anneal/control/space`         | `f`      | Sets space. Maps to `setParam('space', value)`                            |
| `/anneal/control/volume`        | `f`      | Sets master volume. Maps to `setParam('volume', value)`                   |
| `/anneal/control/engine`        | `s`      | Sets active engine ID string. Maps to `setEngine(value)`                  |
| `/anneal/control/session/start` | (none)   | Starts active audio session. Maps to `startSession()`                     |
| `/anneal/control/session/stop`  | (none)   | Fades and stops active audio session. Maps to `stopSession()`             |

#### Engine-Specific Control Actions

Mapped under `/anneal/control/engine_params/<engine>/<param>`:

- **FM**: `/anneal/control/engine_params/fm/<param>` (`f`) -> Sets `modRatio`, `modIndex`, `feedback`.
- **Granular**: `/anneal/control/engine_params/granular/<param>` (`f` or `s`) -> Sets `source`, `size`, `density`, `posJitter`, `pitchJitter`, `posCenter`.
- **Physical**: `/anneal/control/engine_params/physical/<param>` (`f`) -> Sets `model`, `excitationLevel`, `damping`, `brightness`, `reed`, `inharm`.
- **Pulse**: `/anneal/control/engine_params/pulse/<param>` (`f`) -> Sets `density`, `accent`, `tone`, `swing`, `humanize`.

## 2. WebSocket Bridge Protocol Design

Browser applications lack native UDP capability. We bridge native UDP and the browser client using a lightweight JSON-RPC styled frame over standard WebSockets (`ws://localhost:8766`).

### JSON Wire Frame

Both client and bridge helper exchange structured text frames:

```json
{
  "address": "/anneal/control/root",
  "args": [130.81]
}
```

- **`address`**: Mandatory string. Must match the exact canonical OSC path rules.
- **`args`**: Mandatory array. Values must correspond to the target parameter type tags. Specifying an empty array `[]` represents a parameterless bang.

## 3. Standalone Bridge Tool Architecture & Distribution Plan

The bridge tool is a Node-based standalone daemon that translates packets bidirectional between native UDP and WebSocket connections.

```
[ SuperCollider / Reaper ] (UDP Port 9000) <----+
                                                | (OSC UDP)
                                                v
                                     [ tools/osc-bridge ]
                                                ^
                                                | (JSON-over-WS)
                                                v
[ Web Browser (AnnealMusic Client) ] (WS Port 8766)
```

- **UDP Server**: Binds locally to UDP port `8765` to listen for control packets from other software.
- **UDP Client**: Forwards state broadcasts from the WebSocket client to UDP port `9000` (default target, fully configurable) on localhost or specified remote hosts.
- **WebSocket Server**: Binds to port `8766` on localhost, serving JSON wire frames.
- **CLI Commands & Flags**:
  - `--udp-in <port>`: Incoming UDP port (default: `8765`).
  - `--udp-out <port>`: Outgoing target UDP port (default: `9000`).
  - `--udp-host <host>`: Target UDP host address (default: `127.0.0.1`).
  - `--ws-port <port>`: WebSocket listen port (default: `8766`).
  - `--host <ip>`: Local interface IP to bind on (default: `127.0.0.1`).
  - `--filter <rules>`: Path to a JSON configuration of active filters.
  - `--log-level <info|warn|error|debug>`: Log verbosity (default: `info`).
- **Distribution Plan**:
  - Available via npm: `npm install -g annealmusic-osc-bridge`
  - Pre-compiled single-file binaries for macOS, Linux, and Windows built natively via Bun or `pkg` and distributed on the GitHub Releases workspace.

## 4. Mobile Plugin Design

Capacitor web app versions use native hardware UDP sockets instead of the localhost WebSocket bridge, utilizing a unified JS interface wrapper.

```
                  +--------------------------+
                  |    OSC Unified Interface |
                  +-------------+------------+
                                |
             +------------------+------------------+
             |                                     |
             v (Web Platform)                      v (Native Mobile Platforms)
  +--------------------+               +-----------------------------------+
  | WebSocket Bridge   |               | Capacitor Native Plugin           |
  | (tools/osc-bridge) |               |  - iOS: Network.NWConnection      |
  +--------------------+               |  - Android: java.net.Datagram     |
                                       +-----------------------------------+
```

### iOS Native Plugin (`ios/App/App/OSCBridge.swift`)

- Built using the Apple modern `Network` framework (`NWConnection` for outgoing UDP, `NWListener` for incoming UDP).
- Eliminates legacy CocoaAsyncSocket dependencies.
- Handles background thread queue dispatching and serializes incoming packets back to JavaScript using standard Capacitor events.

### Android Native Plugin (`android/app/.../OSCBridgePlugin.java`)

- Implements incoming listener loop inside a background worker thread (`java.lang.Thread` or `ExecutorService`).
- Opens a single `java.net.DatagramSocket` configured to reuse port addresses.
- Decodes binary OSC packets in native Java, wrapping parsed addresses and argument lists inside a standard `com.getcapacitor.JSObject` to trigger JavaScript listeners.

- **Connection Dashboard**:
  - Status indicator (connecting/connected/offline) with glowing, tail-animated HSL badges.

  - Full tree of state and control paths.
  - Interactive "copy to clipboard" button on hover.
- **Real-Time Traffic Log**:
  - Chronological scrollable console of all incoming/outgoing messages.
  - Traffic counters and quick buttons to Clear, Export, or Pause logs.
- **Interactive Rules Editor**:
  - Toggle list of active address filters.
  - Direct control over individual parameter refresh rates (e.g. throttle spectrum to 30Hz, drift to every-change).

## 6. Rate-Limiting & Filter Rule Design

High-frequency outputs (like the 256-bin spectrum or partial arrays) can exhaust UDP packet bandwidth. We incorporate a deterministic client-side filter engine (`OSCFilter.ts`):

- **Dynamic Throttling**:
  - Limits message broadcasts based on a millisecond throttle interval per address.
  - Spectrum streaming defaults to 33ms (~30Hz) limit.
  - Core sculpt parameters default to immediate (`0ms` debounce).
- **Whitelist/Blacklist Filtering**:
  - Individual address paths can be fully enabled or silenced.
  - Filters are stored inside standard per-device persistent preference layers (`localStorage` or Capacitor Preferences).

## 7. Threat Model & Security Specifications

- **Localhost Boundary**:
  - The bridge helper binds strictly to loopback interface `127.0.0.1` by default.
  - A prominent security disclaimer is logged in high-visibility warning color if an administrator binds externally to `0.0.0.0`.
- **Malformed Packet Hardening**:
  - Sockets discard any incoming packet exceeding 65,535 bytes instantly.
  - A hard limit of 64 arguments per packet is enforced; packets failing this constraints are dropped silently.
  - Sockets discard non-conforming OSC address characters immediately to defend against shell injection vector attacks.
- **Client Exhaustion Limit**:
  - Sockets drop incoming packets exceeding a burst rate limit threshold of 100 packets per second from a single host source to block denial-of-service vulnerabilities.

## 9. Key Risks & Mitigations

- **Risk: Bridge Tool Setup Friction**
  - _Mitigation_: Bundle compiled pre-built binaries for easy double-click execution, and provide a clear, one-line copy-paste terminal command (`npx annealmusic-osc-bridge`) directly inside the configuration UI so setup is seamless.
- **Risk: Mobile NAT / Firewalls**
  - _Mitigation_: Document local Wi-Fi port-forwarding instructions and print high-visibility diagnostic helper tips directly on the mobile panel if a UDP socket binding fails or reports connectivity timeouts.
