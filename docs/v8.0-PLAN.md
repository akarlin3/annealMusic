# AnnealMusic v8.0 Audit + Plan (v8.0-PLAN.md)

This document establishes the audit scope, diagnostic tooling, measurement methodologies, and success criteria for **v8.0: Audit + Plan**. This slice performs zero functional code changes (other than non-invasive instrumentation to capture baseline metrics) and delivers the primary audit resources that future v8 slices will execute upon.

---

## 1. Audit Scope per Axis

The audit is structured across four primary axes to evaluate the state of the codebase following the release of v7.7.

### Axis A: Performance

- **Client Cold Start**: Time to interactive (TTI) and core web vitals for the main application (`/index.html`), the learning portal (`/learn.html`), and the research console (`/research.html`).
- **Runtime CPU & Engine Threading**: CPU loads during high partial density sweeps in FM, Granular, and Physical synthesis engines.
- **Mobile Thermal & Memory Footprints**: Background thread audio performance, Capacitor bridge latency, and Web Bluetooth/BLE energy usage.
- **Bundle Weight per Route**: Breakdown of lazy-loaded chunks, external packages (ToneJS, CodeMirror, Yjs), and visualizer assets.

### Axis B: Observability & Safety

- **Dependency Vulnerabilities**: Security vulnerabilities in both Node (`package.json`) and Python (`pyproject.toml`) ecosystems.
- **Threat Model**: Standardized STRIDE analysis mapping input boundaries, file transfers, and sandbox escapes.
- **Silent Failures & Log Coverage**: In-depth inspection of unhandled try/catch blocks in the audio processor, BroadcastChannel bridge, and async background workers.
- **API Surface & Auth Integrity**: Endpoint authentication validation, rate limiters, and data protection levels across `/api/v1/studies`, `/api/v1/reproduce`, and sandbox executors.

### Axis C: Code Health

- **Dependency Upgrades**: Deprecated APIs, old compiler versions, and compatibility issues across ESM/CommonJS barriers.
- **Dead-Code Inventory**: Unused TypeScript files, orphaned UI components, dead Redux/Zustand hooks, and obsolete Python modules.
- **Type Strictness**: TypeScript type safety coverage percentage and pyright typechecking gaps.
- **Schema Drift**: Historical analysis of the parameter serialization schemas and database JSONB drift from version v1 through v22.

### Axis D: Refactor Proposals

- Detailed design documents targeting known architectural smells:
  - **Render Path Duplication**: Consolidation of preview/stem/CLI/video/study renderers.
  - **Schema Source-of-Truth**: Unification of schema definitions across client types, validation rules, and Python engines.
  - **Bridge Transport Sprawl**: Decoupling postMessage, WebSocket, and BroadcastChannel configurations into a single transport interface.
  - **Audio Engine Interface**: Evaluating `AnnealEngine` abstractions.
  - **Capacitor Plugin Sprawl**: Clean abstract layer for mobile integrations.
  - **Pyodide optimization**: Lightening the resource foot-print of scientific sandboxes in `/research`.

## 3. Measurement Methodology

1. **Cold Start Times**:
   - Programmatically measure `navigationStart`, `domContentLoadedEventEnd`, and standard `LCP` / `INP` markers.
   - Insert `performance.mark('audio-init-start')` and `performance.mark('audio-init-end')` boundaries to capture exact engine boot overhead.
2. **CPU Profiling**:
   - Execute a standardized 5-minute stress test: Swap FM, Granular, and Physical engines while maintaining a continuous 32-partial Kuramoto coupled sweep.
   - Profile the AudioWorklet thread active CPU usage and main thread rendering time via DevTools.
3. **Bundle Weight Analysis**:
   - Integrate `rollup-plugin-visualizer` in `vite.config.ts`, `vite.config.mobile.ts`, `vite.config.research.ts`, and `vite.config.learn.ts`.
   - Output individual visualizer reports in `docs/v8/measurements/bundle-reports/`.
4. **Vulnerability Scans**:
   - Run `npm audit --json` and `pip-audit --format json` in the terminal to capture JSON summaries.

## 4. Threat Model Document Outline

The Observability and Safety document will incorporate a STRIDE threat model structured as follows:

1. **Information Flow Boundaries**:
   - _Boundary 1: Browser Front-End to Scientific Python Worker Sandbox_
   - _Boundary 2: Native Capacitor Bridge to Core Device Peripherals (BLE, Health)_
   - _Boundary 3: External Client to API router (FastAPI, SQLite/Postgres DB)_
   - _Boundary 4: Live OSC websocket listener_
2. **Threat Vectors Analyzed**:
   - **Spoofing**: Accessing remote studies or modifying audit logs without matching ORCID credentials.
   - **Tampering**: Modifying Laplace-privatized study outputs to de-anonymize patient records.
   - **Repudiation**: Modifying or deleting local files via sandboxed Python scripts without audit logs.
   - **Information Disclosure**: Unauthorized extraction of custom audio buffers from the shared R2 storage pool.
   - **Denial of Service**: Maliciously structured URL parameters locking the main synthesis engine.
   - **Elevation of Privilege**: Circumventing role verification to gain principal investigator (`pi`) access on private studies.

## 5. Prioritization Framework

All audit findings will be scored mathematically to establish a robust backlog for future slices:

$$\text{Priority} = \frac{\text{Severity (1-5)} \times \text{User Impact (1-5)}}{\text{Fix Cost (1-5)} \times \text{Fix Risk (1-5)}}$$

### Rating Scales

- **Severity**: 1 (Cosmetic/Stale) to 5 (Critical leak / crash).
- **User Impact**: 1 (Affects rare research setups) to 5 (Affects all ambient listening paths).
- **Fix Cost**: 1 (Substantial structural rewrite) to 5 (Trivial one-line fix).
- **Fix Risk**: 1 (Highly critical DSP path) to 5 (Isolated config update).

_Note: Cost and Risk are inverted so that high cost/risk results in a lower priority score, favoring high-impact, easy-to-fix items first._

## 6. Success-Criteria Templates per Slice

Each subsequent v8 slice is held to these exact metrics:

### v8.1 Code Health

- **Type Coverage**: `>= 95%` strict type rating inside `src/`.
- **Dead-Code Reduction**: 100% removal of files and exports flagged by `knip` as completely unused.
- **Dependency Upgrades**: All outdated core dependencies mapped without breaking ESM constraints.

### v8.2 Performance

- **Cold Start Time**: Main application TTI `< 1.8s` on low-power platforms.
- **Bundle Size**: Embed widget route strictly `< 25 KB` gzipped.
- **Worklet Thread**: Total processor computation time `< 2.5ms` per audio process callback.

### v8.3 Observability + Safety

- **Vulnerabilities**: Zero critical/high CVEs inside Node and Python lock files.
- **Auth & Integrity**: 100% coverage of security middleware on FastAPI routes.

### v8.4 Refactor

- **Consolidation**: Single shared audio-render pipeline supporting preview, export, video, and CLI execution.
