# Plan: AnnealMusic v8.3 · Observability + Safety

This plan implements the comprehensive reliability and security updates specified in the v8.0 system audit (`docs/v8/AUDIT_OBSERVABILITY_SAFETY.md`). It spans both the React frontend client and the FastAPI backend server, with a core focus on preserving user privacy while enabling robust error visibility and platform safety.

---

## 1. Error Reporting Choice

### Telemetry Target

We will implement a custom, rate-limited telemetry ingestion endpoint `/api/v1/observability/crash-reports` directly in our FastAPI server.

- **Why this choice:** Instead of utilizing a heavy external third-party SaaS (like sentry.io) or hosting a complex standalone Sentry/GlitchTip container, our own server is capable of serving as the self-hosted Sentry/GlitchTip-compatible receiver.
- **Operator Visibility:** The server will log these reports in structured JSON formatted lines under the logger `"observability"` with level `ERROR`. These naturally flow into the operator's central logging pipeline, keeping them secure and fully local.

---

## 2. Anonymization Filters

Telemetry is strictly **opt-in** and disabled by default. When active, it uses aggressive client-side filters:

1. **User Agent Scrubbing:** Translate precise user-agents to broad classes (e.g., "Safari/macOS" or "Chrome/Android") using a regex helper.
2. **URL Sanitization:** Remove all dynamic parameters, query params, and credentials. Replace active path patterns (e.g. `/p/some-slug-123` with `/p/:slug`, `/jam/session-id` with `/jam/:id`).
3. **Strict State Isolation:** Exclude all database user IDs, auth tokens, email addresses, custom patch parameters, biosignal streams, or listening histories.

## 4. Threat Model Priorities

Based on `docs/v8/AUDIT_OBSERVABILITY_SAFETY.md`, the priority vectors addressed in v8.3 are:

1. **XSS Sanitization:** Audit and secure every rendering boundary that processes user-supplied text (such as visual pacer params, custom sonification mappings, and lesson reflection text).
2. **SlowAPI Rate Limiting:** Implement strict IP-based rate limiting on sensitive routes (`/api/v1/studies/orcid-verify` and `/api/v1/gallery/search`).
3. **Pyodide Sandboxing:** Enforce AST validation and standard RAM ceilings (`MAX_MEM = 512MB`) on Python executions.
4. **CSP Configuration:** Enforce highly secure default-src, frame-ancestors, and mixed-content headers.

## 7. Risks & Mitigation

- **Telemetry Privacy Gaps:** Addressed by using a local telemetry endpoint, doing strict frontend filtering, and requiring explicit checkbox consent.
- **Performance Degradation from Observability:** Addressed by making log tracking async and lightweight inside Starlette middleware.
