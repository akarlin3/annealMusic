# AnnealMusic v9.0 — Top-Level Modes Implementation Plan

This plan outlines the architecture, layout, design, and transition paths for elevating **Mode** from a nested Sketch setting to a **top-level navigation primitive**.

We define three key modes:

1. **Meditation Mode**: For focus, mindfulness, breathing, and deep listening.
2. **Musician Mode**: For creative sculpting, MIDI control, looping, and collaboration.
3. **Researcher Mode**: For scientific sonification, scripting, and psychoacoustic studies.

---

## 1. Mode Definitions & Per-Mode Surface Lists

We define the default landing routes and the visible (foregrounded) vs. hidden components for each mode:

### 🧘 Meditation Mode

- **Landing Route**: `/listen` (the curated session library)
- **Primary Sculpting Interface**: Main creative app, locked/defaulted to **Drone view** (`useParamStore.mode === 'drone'`).
- **Foregrounded Surfaces**:
  - `/listen` (Curated Library)
  - `/timer` (Meditation Timer + Bells + Breath Pacing)
  - `/listening/:slug` (Listening Sessions)
  - `/me/sessions` (Personal listening history)
  - `/learn` (Lessons tagged for meditation/mindfulness)
- **Hidden by Default**:
  - Live collaboration (`/jam/:id`), Loop pedal, Voice inputs, MIDI settings.
  - Public patch gallery (`/gallery`), social feeds (`/feed`).
  - Timeline/Arrangement views (`/piece`), AI generator panel.
  - Biofeedback inputs, clinical studies, datalogging, script console.

### 🎹 Musician Mode

- **Landing Route**: `/` (the main creative app - existing v0.1+ Sketch UI)
- **Primary Sculpting Interface**: Main creative app, with full access to sub-modes (**Sketch / Compose / Drone**).
- **Foregrounded Surfaces**:
  - `/` (Main app workspace)
  - `/gallery` (Community patches gallery)
  - `/feed` (Social activity feed)
  - `/piece` (Timeline, pieces, and arrangement)
  - `/midi` (MIDI routing dashboard)
  - `/me/sessions` (User recording and session history)
  - `/learn` (Lessons tagged for sound design, composition, and production)
  - Header items: Record controls, Collab controls, AI Generate, Export Stems, Presets panel.
- **Hidden by Default**:
  - Curated library `/listen` (accessible via direct URL).
  - CLI tooling, OSC server, Python scripting console, clinical protocols, datalogger.

### 🔬 Researcher Mode

- **Landing Route**: `/research.html` (the primary research console)
- **Primary Sculpting Interface**: The Research Console (`/research.html`) with embedded telemetry, spectrum visualization, and custom widgets.
- **Foregrounded Surfaces**:
  - `/research.html` (Scripting, datalogging, OSC, telemetry, and experiment builder)
  - `/clinical/:slug` (Clinical subject trial runner)
  - `/experiment/:slug` (Psychoacoustic experiment runner)
  - `/reproduce` (Study reproducer page)
  - `/learn` (Lessons tagged for crossover music + science track)
- **Hidden by Default**:
  - Gallery, feed, timelines, collaboration controls, curated libraries, and consumer breathing pacing interfaces.

## 2. Mode Persistence & First-Time Picker UX

### Persistence Layer

- Sticky per-device storage utilizing the shared `appStorage` (`src/platform/storage.ts`), which transparently selects local storage (web) or Capacitor Preferences (native iOS/Android).
- **Storage Key**: `am_app_mode`
- **First-Time Detection**: If `appStorage.get('am_app_mode')` returns `null`, the user is considered a first-time visitor.
- **Default for existing/returning users**: `"musician"` (prevents disruption to returning workflows).

### First-Time Picker UX

- A beautiful, immersive, glassmorphic modal overlay mounted on the root route `/` if no mode is persisted.
- **Visuals**: CURATED harmonious, dark-theme layout with three large, interactive cards side-by-side (or stacked on mobile).
  - **Meditation card**: Elegant breathing wave micro-animation, calming pastel colors (indigo/violet), text: _"Focus, settle, breathe. Enter a minimalist ambient sanctuary."_
  - **Musician card**: Rotating wave/knob graphics, vibrant creative colors (amber/gold), text: _"Create, capture, collaborate. Access multi-track loop tools and MIDI controls."_
  - **Researcher card**: Matrix/graph visualization, technological blue/stone palette, text: _"Analyze, script, experiment. Leverage OSC bridges, Python REPLs, and clinical workflows."_
- **Action**: Clicking a card instantly saves that mode to storage and performs a smooth, instant transition/navigation to that mode's landing route.
- **Dismissal**: A clean "Skip to Musician Sandbox" option at the bottom ensures zero friction for anyone wanting to jump straight in.

## 3. Mode Switcher UI & Keyboard Shortcut

### Mode Switcher Component (`ModeSwitcher.tsx`)

- A premium, segmented tab control displayed prominently in the app header (or top-right of every sub-application).
- Segment labels: **Meditation** (or `🧘`), **Musician** (or `🎹`), **Researcher** (or `🔬`).
- Micro-animations: Smooth sliding background pill behind the active mode segment.
- Switching triggers:
  1. Instant update to local storage/Capacitor Preferences.
  2. Smooth update to the `ModeContext` state.
  3. Immediate navigation to the target mode's landing route (e.g. `/listen`, `/`, `/research.html`).

### Keyboard Shortcut

- Listening for `Shift+M` at the window level.
- Cycles modes in sequence: `Meditation` → `Musician` → `Researcher` → `Meditation`.
- Pressing `Shift+M` flashes a brief, elegant toast/overlay notifying the user of the active mode change and navigates instantly.

## 5. Edge Cases & Deferrals

- **Shared Links / External URLs**: If a Meditation-mode user clicks a shared link to `/gallery` or a collab session `/jam/123`, the app **must render the page normally** without blocking access. Mode is a soft constraint.
- **Header Sync**: We will inject `ModeContext` into the root of each app so the header remains unified and responsive.
- **Server Sync**: Cross-device mode synchronization via a server-side `preferred_mode` column is **deferred post-v9.0**.

---

## 6. Mobile Considerations

- On native mobile platforms (Capacitor apps), screen space is at a premium.
- The `ModeSwitcher` will scale gracefully:
  - Large screens: Full labels next to icons (e.g. `🧘 Meditation`).
  - Mobile screens: Sleek icon-only segmented controls (`🧘` / `🎹` / `🔬`) placed at the top-right of the navigation bar or header menu to save valuable visual real estate.

## 7. Risks & Mitigation Plan

| Risk                           | Impact | Mitigation                                                                                                                                                                                                                                          |
| :----------------------------- | :----- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Deep Bookmark Disruption**   | High   | Returning users with existing bookmarks bypass the landing picker by checking if a direct sub-path (like `/gallery`) is loaded. Picker only displays on `/`.                                                                                        |
| **Accidental Mode Switching**  | Medium | The header switcher is extremely visible, letting users recover instantly. Direct URL paths remain fully functional.                                                                                                                                |
| **Separate HTML Bundles Sync** | High   | The distinct single-page entry points (`index.html`, `research.html`, `learn.html`) do not share React routing. We use `appStorage` to guarantee consistency: when a sub-app boots, it reads the current mode from storage and renders accordingly. |
