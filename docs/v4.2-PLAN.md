# project plan - AnnealMusic v4.2 · Drone Mode

This plan introduces a new top-level **Drone Mode** in AnnealMusic, offering a constrained, minimalist, and deeply meditative UI on top of the existing audio engine.

---

## 1. Mode Discriminator Data Model + URL v18 + Migration Plan

### 1.1 Data Model Changes

We will modify two tables: `patches` and `listening_sessions`.

- **`patches` table**:
  - The `mode` column already exists in SQLAlchemy (defaulting to `'open'`).
  - We will alter the default/server_default of this column to be `'sketch'`.
  - We will run a data migration to update all existing rows' `mode` values from `'open'` / `'arc'` to `'sketch'`.
  - In SQLAlchemy model `Patch`, we set the default/server default to `'sketch'`, and document its new enum values: `'sketch' | 'drone'`.

- **`listening_sessions` table**:
  - Split column approach for referential integrity:
    - We will add `patch_id` column as UUID referencing `patches(id)` with ON DELETE CASCADE.
    - We will add check constraint `ls_source_one_of` ensuring exactly one of `piece_id` or `patch_id` is set: `(piece_id IS NOT NULL) != (patch_id IS NOT NULL)`.
    - We will update the `piece_id` foreign key relation to be ON DELETE CASCADE for total delete consistency.
  - Social items / feed schemas will be updated to display patches or pieces wrapped in listening sessions dynamically.

### 1.2 URL Schema v18

We will bump `SCHEMA_VERSION` in `src/share/schema.ts` from `17` to `18`.

- In v18, URL state payloads support an optional `mode` query parameter (e.g. `mode=drone` or `mode=sketch`).
- If `mode` is missing, it defaults to `'sketch'` for full backward compatibility with older URLs.
- Decoded state is passed to the store on hydration and correctly sets the active UI mode.

## 2. Drone Constraint Enforcement

Enforced at the **orchestrator** level in `src/audio/orchestrator.ts`:

- We will define a `ModeConstraints` adapter in `src/drone/ModeConstraints.ts` which takes the target parameters and returns clamped/overridden versions:
  - `drift` is clamped to ≤ 0.2.
  - `coupling` is forced to ≥ 0.6.
  - `spread` is overridden to exactly 1.0.
  - `space` (reverb) is forced to ≥ 0.5.
- If in drone mode, the orchestrator:
  - Forces the active UI mode to `'drone'`.
  - Runs any parameter changes (via `setSharedParams`) through the `ModeConstraints` adapter.
  - Disables the pulse engine entirely.
  - Ignores the notation track entirely.
  - Disables arc mode entirely (starts only in Open/Continuous session).

## 3. Drone-Suitable Engine Subset and per-Engine Drone-Mode Defaults

We will create `src/drone/droneEngines.ts` to manage subset curations:

- **Engine Subset**: Only `sine`, `granular`, and `physical` engines are drone-suitable.
- **Granular Engine Curated Sources**: Only sustained atmospheric pad sources are allowed in the selector:
  - `glasspad` (0)
  - `bowedmetal` (1)
  - `tapeorgan` (2)
  - `deepdrone` (4)
  - `choirair` (5)
  - `warmtape` (7)
- **Physical Engine Curated Models**: Only continuous sustain models are allowed in the selector:
  - `string` (0)
  - `bowed` (4)
  - `mallet` (5)
  - `bell` (7)

## 4. Drone UI Design

A minimalist, high-contrast, beautiful interface built in `src/drone/DroneView.tsx`:

- **Centerpiece Visualizer**: Uses the WebGL or Canvas renderer with `isCalm={true}` (reduces movement speed by 55%, sets inner halo to deep warm amber glow, and creates slow breathing organic patterns).
- **Single Large Root Note Indicator**: Renders the active fundamental in bold beautiful text: e.g., `A2 · 110 Hz` using the active tuning system.
- **Three Sliders Only**:
  - Fundamental (Root Frequency).
  - Density (curated to 1, 3, or 5 partials only).
  - Brightness.
- **Compact Engine Picker**: Curated options for `Sine`, `Granular`, `Physical`.
- **Compact Tuning Picker**: Full v4.1 scale selections in a beautiful dropdown/segmented selector.
- **Begin / Settle Button**: Calming play/stop button.

## 5. Listening Session Wrapping a Drone

- Creatively publish drone sessions: a user creates a drone in the synthesizer, saves it (saved as `mode: 'drone'`), and wraps it directly in a Listening Session.
- The Listening Session creation flow lists the user's saved drones alongside composed pieces.
- For drone-based listening sessions, target durations are derived from the selected `length_category`:
  - `'short'`: 10 minutes (600k ms)
  - `'medium'`: 20 minutes (1.2m ms)
  - `'long'`: 30 minutes (1.8m ms)
  - `'extended'`: 60 minutes (3.6m ms)
- Opening/closing bells are appended as +4000ms chimes respectively.

---

## 6. Offline Render Duration Controls for Drones

- In `src/export/ExportDialog.tsx`, when rendering a drone patch offline, we present a duration selection dropdown (defaulting to 30 minutes, with options for 10m, 20m, 30m, 60m).
- Offline rendering proceeds deterministically using existing capture streams and v1.5 render engines.

## 8. Migration Safety

- **Data Integrity**: Safe SQLite & Postgres migrations using Alembic. Existing values `'open'` / `'arc'` in the `patches.mode` column are updated to `'sketch'`.
- **FK Constraints**: Old `piece_id` is renamed/moved to `piece_id` and `patch_id` with `ls_source_one_of` check constraint. Existing listening session references remain intact.

---

## 9. Risks and Mitigations

- **Discoverability of Mode Switch**: Swapping modes is made highly prominent in the header using elegant micro-animations.
- **Underwhelming Pitch/Sketch UI**: If a drone patch is loaded in the full Sketch UI, all parameters are unlocked, but it starts with its drone shape. Help descriptions explicitly clarify this transition.
