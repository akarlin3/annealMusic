# Implementation Plan — v1.4 Mobile Shell (Capacitor, iOS + Android)

This implementation plan details the technical architecture, configuration, code layout, and verification steps to package the existing Anneal Ambiance web app into native iOS and Android shells using **Capacitor 6.x**.

We maintain a **single codebase** producing three deployment targets (Web SPA, iOS app, and Android app) without duplicating business logic, audio engines, or persistence layers. Platform-specific hooks are abstracted behind a sleek, statically-typed `PlatformBridge`.

---

## User Review Required

Please review the following key decisions and architectural elements:

> [!IMPORTANT]
> **Custom Native Audio Session Shim vs. Community Plugin**
> We reject `@capacitor-community/background-mode` because it is an oversized, complex plugin designed to prevent the OS from suspending the webview entirely. Instead, we implement a **custom native shim** (a single Swift file for iOS, a single Kotlin file for Android).
>
> - **iOS Objective**: Configure `AVAudioSession` to category `.playback` (exclusive, not ambient) so background audio functions natively when the screen locks, and capture interruption notifications (calls, Siri, other playback).
> - **Android Objective**: Manage `AudioFocusRequest` with `AUDIOFOCUS_GAIN` and map phone/headphone events.
>   This approach is extremely light, has zero third-party dependencies, and carries virtually zero App Store rejection risk.

> [!WARNING]
> **Magic Link Auth Cookie Sharing & Session Persistence**
> The mobile shell uses cross-origin API requests (origin `capacitor://localhost` or `http://localhost` calling `https://api.annealmusic.org`). Traditional WebView cookies face severe CORS restrictions.
>
> - **Solution**: We enable `@capacitor/cookies` and `@capacitor/http` in `capacitor.config.ts`. This overrides the global `fetch` API inside the WebView.
> - **Mechanism**: When a magic link is verified, the server sets an `am_session` HTTP-only cookie. Capacitor's native layer intercepts the `Set-Cookie` header, persists it in the native cookie jar, and automatically injects it in subsequent WebView fetches. This makes magic link auth round-trips work flawlessly.

const STORAGE_KEY = 'am_anon_id';

export const webBridge: PlatformBridge = {
  getPlatform: () => 'web',
  getPersistedAnonId: async () => localStorage.getItem(STORAGE_KEY),

        name: 'microphone' as PermissionName,
      });
      return state.state as 'granted' | 'denied' | 'prompt';
    } catch {
      return 'prompt';
    }
  },
  onAudioInterruption: () => {
    // No-op on standard web
    return () => {};
  },
  openAppSettings: async () => {
    // No-op
  },
};

#### [NEW] [capacitor.ts](file:///Users/averykarlin/annealMusic/src/platform/capacitor.ts)

Native implementations targeting iOS and Android WebView APIs.

```typescript
import { Capacitor } from '@capacitor/core';
import { Preferences } from '@capacitor/preferences';
import { PlatformBridge } from './types';

const STORAGE_KEY = 'am_anon_id';

export const capacitorBridge: PlatformBridge = {
  getPlatform: () => (Capacitor.getPlatform() === 'ios' ? 'ios' : 'android'),
  getPersistedAnonId: async () => {
    const { value } = await Preferences.get({ key: STORAGE_KEY });
    return value;
  },
  setPersistedAnonId: async (id) => {
    await Preferences.set({ key: STORAGE_KEY, value: id });
  },
  clearPersistedAnonId: async () => {
    await Preferences.remove({ key: STORAGE_KEY });
  },
  requestMicPermission: async () => {
    // Direct getUserMedia permissions in WebView will trigger native prompts,
    // but we can query or request permission status natively for smooth UI feedback.
    if (Capacitor.getPlatform() === 'ios') {
      // iOS permission query
      const result = await requestNativeMicrophonePermission();
      return result;
    } else {
      // Android permission query
      const result = await requestNativeMicrophonePermission();
      return result;
    }
  },
  onAudioInterruption: (handler) => {
    const listener = (event: any) => {
      handler(event.detail.type); // 'begin' or 'end'
    };
    window.addEventListener('am_audio_interruption', listener);
    return () => {
      window.removeEventListener('am_audio_interruption', listener);
    };
  },
  openAppSettings: async () => {
    // Redirect to native app settings
    window.open('app-settings://');
  },
};

// Helper for native permission integration
async function requestNativeMicrophonePermission(): Promise<
  'granted' | 'denied' | 'prompt'
> {
  // Bridge interface details...
  return 'granted';
}
```

### 4. Audio Session & Background Audio Handling

To support background playback (continuing audio on iOS screen lock and app switching) and handle interruptions (like phone calls, headphone unplugging, or other music apps):

#### iOS `AVAudioSession` Setup (Swift Plugin)

We add a custom background audio setup in our `ios/App/App/AppDelegate.swift` or a dedicated Capacitor plugin:

```swift
import AVFoundation
import Capacitor

@objc(AMAudioSessionPlugin)
public class AMAudioSessionPlugin: CAPPlugin {
  @objc func initSession(_ call: CAPPluginCall) {
    let session = AVAudioSession.sharedInstance()
    do {
      // Playback category: enables background audio, respects the ring/silent switch, doesn't suspend.
      try session.setCategory(.playback, mode: .default, options: [])
      try session.setActive(true)

      // Observe interruptions
      NotificationCenter.default.addObserver(
        self,
        selector: #selector(handleInterruption),
        name: AVAudioSession.interruptionNotification,
        object: nil
      )

      call.resolve()
    } catch {
      call.reject("Failed to configure AVAudioSession: \(error.localizedDescription)")
    }
  }

  @objc func handleInterruption(notification: Notification) {
    guard let userInfo = notification.userInfo,
          let typeValue = userInfo[AVAudioSessionInterruptionTypeKey] as? UInt,
          let type = AVAudioSession.InterruptionType(rawValue: typeValue) else { return }

    if type == .began {
      // Dispatches event to WebView JavaScript
      self.bridge?.triggerJSEvent(eventName: "am_audio_interruption", target: "window", data: "{\"type\": \"begin\"}")
    } else if type == .ended {
      if let optionsValue = userInfo[AVAudioSessionInterruptionOptionKey] as? UInt {
        let options = AVAudioSession.InterruptionOptions(rawValue: optionsValue)
        if options.contains(.shouldResume) {
          self.bridge?.triggerJSEvent(eventName: "am_audio_interruption", target: "window", data: "{\"type\": \"end\"}")
        }
      }
    }
  }
}
```

#### Android `AudioManager` Setup (Kotlin Plugin)

In `android/app/src/main/java/org/averykarlin/anneal/MainActivity.kt`, we wire up Android's `AudioFocusRequest`:

```kotlin
import android.media.AudioManager
import android.media.AudioAttributes
import android.media.AudioFocusRequest
import android.os.Build

class MainActivity : BridgeActivity() {
    private lateinit var audioManager: AudioManager
    private var focusRequest: AudioFocusRequest? = null

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        audioManager = getSystemService(AUDIO_SERVICE) as AudioManager
        requestAudioFocus()
    }

    private fun requestAudioFocus() {
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
            val playbackAttributes = AudioAttributes.Builder()
                .setUsage(AudioAttributes.USAGE_MEDIA)
                .setContentType(AudioAttributes.CONTENT_TYPE_MUSIC)
                .build()

            focusRequest = AudioFocusRequest.Builder(AudioManager.AUDIOFOCUS_GAIN)
                .setAudioAttributes(playbackAttributes)
                .setAcceptsDelayedFocusGain(true)
                .setOnAudioFocusChangeListener { focusChange ->
                    when (focusChange) {
                        AudioManager.AUDIOFOCUS_LOSS, AudioManager.AUDIOFOCUS_LOSS_TRANSIENT -> {
                            // Notify web context to pause audio gracefully
                            bridge.triggerJSEvent("am_audio_interruption", "window", "{\"type\": \"begin\"}")
                        }
                        AudioManager.AUDIOFOCUS_GAIN -> {
                            // Notify web context to resume audio gracefully
                            bridge.triggerJSEvent("am_audio_interruption", "window", "{\"type\": \"end\"}")
                        }
                    }
                }
                .build()

            audioManager.requestAudioFocus(focusRequest!!)
        }
    }
}
```

### 5. Deep Link & Magic Link Routing

To allow opening saved patches (`/p/<slug>`), recordings (`/r/<slug>`), profiles (`/u/<account_id>`), and handling email authentication magic links (`/auth/email/verify?token=...`):

#### Universal Links (iOS) & App Links (Android) Integration

1. **Host Site Assets**: We place the metadata files directly in our Vite project’s static assets path `public/.well-known/` directory.
   - **`public/.well-known/apple-app-site-association`** (iOS)
   - **`public/.well-known/assetlinks.json`** (Android)
2. **Apple App Association Spec (`apple-app-site-association`)**:
   ```json
   {
     "applinks": {
       "apps": [],
       "details": [
         {
           "appID": "8K6V3Y82AA.org.averykarlin.anneal",
           "paths": ["/p/*", "/r/*", "/u/*", "/auth/email/verify*"]
         }
       ]
     }
   }
   ```
3. **Android App Association Spec (`assetlinks.json`)**:
   ```json
   [
     {
       "relation": ["delegate_permission/common.handle_all_urls"],
       "target": {
         "namespace": "android_app",
         "package_name": "org.averykarlin.anneal",
         "sha256_cert_fingerprints": [
           "FA:C6:17:45:DC:9B:03:7E:21:..." // Release certificate SHA256
         ]
       }
     }
   ]
   ```

#### Magic Link Round-Trip cookie sharing:

Because we enabled `CapacitorCookies` and `CapacitorHttp`:

1. When a user clicks a verification magic link in their native mail app (e.g. `https://anneal.averykarlin.org/auth/email/verify?token=123`), the Universal/App Link launches the native Anneal Ambiance app shell.
2. The app shell intercepts this link using the `@capacitor/app` plugin's `appUrlOpen` listener:

   ```typescript
   import { App } from '@capacitor/app';
   import { useNavigate } from 'react-router-dom';

   App.addListener('appUrlOpen', (data) => {
     // Extract URL pathname + query parameters
     const url = new URL(data.url);
     const relativePath = url.pathname + url.search;

     // Route directly inside React Router
     if (url.pathname.startsWith('/auth/email/verify')) {
       // Navigate to the verification page, which performs a transparent fetch request
       // against the API. The native Cookie store automatically captures and holds
       // the resulting HTTP-only session cookie!
       window.location.href = relativePath;
     } else {
       // Normal in-app routing
       navigate(relativePath);
     }
   });
   ```

3. Subsequent `fetch` calls made by the WebView will transparently include the session cookie, keeping the user securely signed in across restarts.

### 7. Versioning Strategy

We maintain the **single source of truth** in `package.json`'s `"version"`.

#### Android Auto-Versioning (`android/app/build.gradle`)

We parse `package.json` directly within Gradle to inject versions dynamically:

```groovy
import groovy.json.JsonSlurper

def getAppVersion() {
    def jsonFile = file('../../package.json')
    def parsedJson = new JsonSlurper().parseText(jsonFile.text)
    return parsedJson.version
}

def getGitCommitCount() {
    try {
        def stdout = new ByteArrayOutputStream()
        exec {
            commandLine 'git', 'rev-list', '--count', 'HEAD'
            standardOutput = stdout
        }
        return Integer.parseInt(stdout.toString().trim())
    } catch (ignored) {
        return 1
    }
}

android {
    defaultConfig {
        versionCode getGitCommitCount()
        versionName getAppVersion()
    }
}
```

## Risks, Mitigation & Tradeoffs

### 1. WKWebView vs. Chrome AudioWorklet Performance

- **Risk**: WKWebView on iOS has strict memory limits and power throttling, which can degrade high-polyphony Physical-modeling worklet performance.
- **Mitigation**: We target iOS 15+ as a baseline, which utilizes modern, highly optimized AudioWorklet threads. We perform local stress testing on a range of hardware (from iPhone XS to iPhone 15 Pro) to tune safety limits if necessary.

### 2. Sandbox Storage Quotas

- **Risk**: The WebView's `localStorage` quota (~5MB) could easily be exhausted by uploading custom user sources or long loops.
- **Mitigation**: Anonymous IDs are persisted in `@capacitor/preferences` (backed by native user defaults/preferences, which has no storage quotas). User-uploaded files (v1.2) are saved in standard object storage on the server, not locally, mitigating local database bloat.

### 3. Apple App Review Creative App Constraints

- **Risk**: Apple sometimes rejects pure web wraps under App Store Review Guideline 4.2 (Minimum Functionality) if they lack native feeling.
- **Mitigation**: We utilize customized Instrument Serif font render styling, premium dark layouts, fully fluid multi-touch visualizers, deep linking integrations, and proper background audio. This ensures a native look and feel.

## Tradeoffs Flagged

> [!CAUTION]
> **No Lock-Screen Metadata / Media Controls in v1.4**
> We deliberately defer integrating `MediaSession` API and locking control buttons (e.g. Play/Pause/Skip) on the lock screen. The app runs pure audio background playback. If required, lock-screen media controls will be added in v1.5 using a specialized media integration plugin.

> [!NOTE]
> **No iOS CI Automation**
> We compromise on automated iOS builds to avoid the high monthly overhead of custom Mac Runners. Local Xcode compiling is sufficient for v1.4 internal Flight testing.
