# AnnealMusic v7.1 · Sonification Engine · Project Plan

This document establishes the comprehensive project plan, technical specifications, and system design for the purpose-built **Sonification Engine** (AnnealMusic v7.1). It satisfies the hard-stop prerequisite of Checkpoint 0 before commencing any codebase modifications.

---

## 1. Final Data Model & URL Schema (v21)

### 1.1 Database Schema

The database model represents a **Sonification** as a first-class shared and publishable research artifact.

```sql
CREATE TABLE sonifications (
  id              UUID PRIMARY KEY,
  user_id         UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  schema_ver      INTEGER NOT NULL,
  title           TEXT,
  description     TEXT,
  base_state      JSONB NOT NULL,           -- patch state used as the baseline
  mapping_spec    JSONB NOT NULL,           -- the full mapping rules
  source_files    JSONB NOT NULL DEFAULT '[]'::jsonb,  -- file references (storage keys)
  duration_ms     INTEGER,
  visibility      TEXT NOT NULL DEFAULT 'unlisted',
  short_slug      TEXT UNIQUE,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_sonifications_user ON sonifications(user_id);
```

Each entry in `source_files` JSONB array contains:

- `storage_key`: S3/R2 path (e.g. `sonifications/<user_id>/<sonification_id>/<file_uuid>.csv`).
- `filename`: Original name of the uploaded data file.
- `row_count`: Number of data records.
- `columns`: Array of channel headers parsed from the file.

### 1.2 URL Schema v21

We bump the schema version to `21`. Shared URL states carry:

- `kind=sonification`
- `base`: Encoded URL patch payload matching the baseline patch state.
- `spec`: URI-encoded JSON string containing the mapping spec.
- `sources`: Reference details of loaded files.
- `dur`: Total duration in milliseconds.

Existing patches (v1 to v20) continue to parse seamlessly. If no `kind` is present, it is parsed as a legacy patch.

## 2. Mapping Spec Schema

The `mapping_spec` defines a list of mapping rules. A rule connects a source data channel to a target synthesizer parameter.

```json
{
  "mappings": [
    {
      "source_channel": "temperature",
      "target_param": "brightness",
      "transform": {
        "type": "linear",
        "domain": [0, 100],
        "range": [0, 1],
        "axis": "linear",
        "clip": true
      },
      "calibration": {
        "min_perceptual": 0.2,
        "max_perceptual": 0.85
      }
    }
  ]
}
```

- **source_channel**: String column name from the source stream.
- **target_param**: Shared parameters (`rootFreq`, `spread`, `density`, `coupling`, `drift`, `brightness`, `space`) or engine-specific faders (e.g. `fm.modIndex`, `gr.posJitter`).
- **transform**: Mathematical scaling parameters.
- **calibration**: Perceptually scaled output constraints.

## 3. Transform Implementations

Transforms map a raw float $x$ from a data column into a normalized parameter value $y \in [0, 1]$ (or parameter-specific range).

1. **`linear(domain, range)`**:
   $$y = R_0 + \frac{x - D_0}{D_1 - D_0} \cdot (R_1 - R_0)$$
   If `clip` is active, $y$ is clamped to $[R_0, R_1]$.

2. **`log(domain, range)`**:
   $$y = R_0 + \frac{\log(x) - \log(D_0)}{\log(D_1) - \log(D_0)} \cdot (R_1 - R_0)$$
   _Constraint:_ $D_0, D_1, x > 0$. Zero or negative values fallback to linear.

3. **`exp(domain, range)`**:
   $$y = R_0 + \frac{e^x - e^{D_0}}{e^{D_1} - e^{D_0}} \cdot (R_1 - R_0)$$

4. **`discrete(value_map)`**:
   Maps categorical/discrete keys directly to configured target values. If $x \notin \text{value\_map}$, it falls back to a default parameter baseline.

5. **`quantile(bins, range)`**:
   Assigns $x$ to $N$ uniform frequency quantiles, then maps bins to stepped intervals in $R$.
   $$\text{bin} = \text{clamp}\left(\left\lfloor \frac{x - D_0}{D_1 - D_0} \cdot B \right\rfloor, 0, B-1\right)$$
   $$y = R_0 + \frac{\text{bin}}{B - 1} \cdot (R_1 - R_0)$$

6. **`pcm(samplerate)`**:
   Direct injection. Stubs for v7.1 (advanced high-frequency voltage inject mapping).

7. **`python(script)`**:
   Placeholder in v7.1. Allows dynamic script evaluation inside Pyodide sandboxes (post-v7.1).

## 4. Source Adapters

Sources act as numeric series iterators. A source resolves a set of key-value pairs at any elapsed time $t$.

### 4.1 FileSource

Parses CSV and JSON data files client-side.

- Holds parsed data as a sorted array of `(timestamp, values)` records.
- **Interpolation:** At any elapsed time $t$, locates surrounding ticks $t_{prev}$ and $t_{next}$. Linearly interpolates faders between them:
  $$v(t) = v_{prev} + \frac{t - t_{prev}}{t_{next} - t_{prev}} \cdot (v_{next} - v_{prev})$$
- Binary Parquet and HDF5 uploads are uploaded to the backend S3/R2 storage, parsed on-demand by the FastAPI server via pandas/pyarrow, and fetched by the client as standardized JSON arrays.

### 4.2 SyntheticSource

Generates mathematical waveforms (e.g. Sine, Triangle, Noise, Random Walk, custom mathematical curves) as data channels. Perfect for prototyping and script evaluations.

### 4.3 LiveSource

Receives real-time ticks over WebSockets or SSE channels. Every received tick pushes an instant parameter modulation event to the orchestrator.

## 5. Calibration Mode Design

Auditory perceptual mapping varies between listeners and faders. Calibration ensures balanced auditory feedback.

- **Auditory Testing:** Next to each mapping, the UI houses "Test Min" and "Test Max" buttons.
- **Synthesis Injection:** Triggering these temporarily overrides the running engine fader to its mapped minimum or maximum limit.
- **Perceptual Tuning:** The user listens to the synthesis outputs and moves the sliders to scale output faders (assigning `min_perceptual` and `max_perceptual` thresholds).
- **Metadata Citation:** The calibration boundaries are saved inside the mapping spec JSON and cited in study reports.

## 6. Playback Architecture in the Orchestrator

The sonification engine operates as a decoupled layer inside the `Orchestrator`, running in parallel with voice inputs and loop pedals:

- **Real-Time Loop:** Runs at 50Hz (every 20ms). Calculates current elapsed session time $t = \text{rate} \cdot (\text{currentTime} - t_0)$. Resolves column values from active adapters, applies transforms, scales through perceptual calibrations, and calls `setSharedParams` and `setEngineParams`.
- **Interpolation:** Targets parameters smoothly using the Web Audio `setTargetAtTime` method, matching fader changes to the 50Hz telemetry frame rate to eliminate click artifacts.
- **Offline Rendering:** Reuses `StemRenderer.ts` sequential contexts. At each checkpoint (50Hz suspension mark), the sonification engine resolves the exact frame variables, driving deterministic multi-track stem outputs.

---

## 7. Performance Budgets

- **Real-time processing:** Live sonification mapping tick evaluation must complete in under **2.0ms** to prevent UI blocks or audio thread glitches.
- **Memory Overhead:** Loaded time-series data sets in-browser are capped at **5MB** (approximately 100,000 rows across 5 columns). Large datasets must be decimated or downsampled server-side or rendered offline.
- **Offline Rendering Speed:** Deterministic render must perform at $\ge 15\times$ speedups (a 5-minute sonification renders in under 20 seconds).
- **Asset Size:** Client code-split bundle limits sonification modules to **120KB** gzipped.

## 9. Risk Assessment & Mitigations

### 9.1 Mapping Complexity vs. UX

- _Risk:_ Researchers find a raw mathematical mapping grid confusing.
- _Mitigation:_ Pre-seed the system with 3 standard "ICAD Recipes" (e.g. Weather sonification, Stock market walk, Sine Sweep) to let users instantly click and hear working mapping systems.

### 9.2 Source File size Limits

- _Risk:_ Massive 50MB time-series files cause browser memory leaks.
- _Mitigation:_ Restrict browser file upload sizes to 5MB. If a file is larger, backend uploads are allowed, and pandas downsamples the data to 20,000 rows before returning it to the client.

### 9.3 Calibration Variability

- _Risk:_ Listener perception shifts across headphones or ambient environments.
- _Mitigation:_ Ensure all calibrations are saved as local and editable mapping constraints, allowing any researcher loading a shared sonification link to recalibrate the system for their own audio setup and save the updated calibration.
