# Implementation Plan — v4.3 Bells & Punctuation

This implementation plan details the architectural and feature design for **v4.3 Bells & Punctuation**. The objective is to replace the synthesized placeholder chime from v4.0 with a high-fidelity curated library of 12 real/synthesized bells, add interval-based schedule punctuation for Listening Sessions, Pieces, and a Standalone Timer, and bump the URL/storage schema version to v19.

---

## User Review Required

> [!IMPORTANT]
> **Data Migration & Backward Compatibility:**
> v4.0's simple `opening_tone` and `closing_tone` boolean columns in `listening_sessions` will be dropped. They will be replaced by a structured `bell_schedule` column storing JSONB arrays. Existing user records with `opening_tone = true` and `closing_tone = true` will automatically migrate to single-bell `at-start` and `at-end` schedule events using the Zen meditation bell (`zen_bell_rin`) as a pristine, authentic default.
>
> **Asset Delivery & Size Target:**
> All 12 bells are synthesized programmatically using high-quality physical modeling algorithms (additive, FM, and waveguide synthesis) inside a Node.js asset generator script. They will be exported to 16-bit mono 48kHz WAV files and compressed to mono Opus 96kbps format. They are released as CC0 public-domain original works, ensuring 100% clean licensing without any external copyright dependencies.

### 2. Schedule Data Model & URL v19

The bell schedule model represents an array of events:
```typescript
export interface BellEvent {
  bellId: string;
  trigger: 'at-start' | 'at-end' | 'at-time' | 'every' | 'at-movement-start' | 'at-movement-end';
  offsetMs?: number; // Used for 'at-start', 'at-end', 'at-time'
  intervalMin?: number; // Used for 'every' (recurring)
  movementIndex?: number; // Used for 'at-movement-start', 'at-movement-end'
  volume: number; // 0..1, default 0.7
}
```

#### URL Schema (v19)
- **Bump version to 19** in `src/share/schema.ts`.
- **Patch/Piece default config:** Add `bell_schedule` serialization/deserialization.
  - Key code: `b.sched` for the serialized JSON string or a compressed format.
  - Serialization: In `encodeListeningSession` and `encodePiece`, we serialize `bell_schedule` into the URL payload.
  - Migration: Older URLs (v18 and lower) containing boolean `openB=1` / `closeB=1` or `opening_tone` / `closing_tone` will auto-migrate.

### 3. Audio Core Integration

- **Graph Routing:**
  - Bells are loaded using `AudioContext.decodeAudioData` (lazy loaded on first use).
  - Bell triggers create an `AudioBufferSourceNode` + `GainNode` and connect **directly to `masterVol`**, post-summation but pre-master gain.
  - This bypasses the main post-fx lowpass `filter` and engine `convolver` (reverb), keeping the bells dry, clean, and present.
- **Sample-Accurate Scheduling:**
  - `BellScheduler` handles scheduling bell triggers using the audio context's `currentTime`.
  - Runs in a regular tick / look-ahead loop (like standard Web Audio schedulers) to schedule events slightly in advance to guarantee jitter-free, sample-accurate timing even under heavy CPU loads.

### 4. Database Schema Migration

We will write a SQLAlchemy model update and Alembic migration `0013_v4_3_bells` to perform the following:
```sql
ALTER TABLE listening_sessions DROP COLUMN opening_tone;
ALTER TABLE listening_sessions DROP COLUMN closing_tone;
ALTER TABLE listening_sessions ADD COLUMN bell_schedule JSONB NOT NULL DEFAULT '[]'::jsonb;
ALTER TABLE pieces ADD COLUMN bell_schedule JSONB NOT NULL DEFAULT '[]'::jsonb;
```
Data migration preserves old user choices:
- `opening_tone = true` $\rightarrow$ `{"bellId": "zen_bell_rin", "trigger": "at-start", "volume": 0.7}`.
- `closing_tone = true` $\rightarrow$ `{"bellId": "zen_bell_rin", "trigger": "at-end", "volume": 0.7}`.

### 5. UI Panels & Standalone Timer

- **BellScheduleEditor Component (`src/listening/BellScheduleEditor.tsx`):**
  - Accordion or inline card sub-panel under "Bells & Punctuation".
  - Preview button next to each bell selection (triggers immediate dry play of the loaded bell sample).
  - Footers displaying descriptions + licensing attributions.
  - Schedule list: Add, delete, and edit bell events (change trigger type, pick bell, adjust volume slider, enter interval minutes or offsets).
  - Visual timeline markers on the session length bar or piece movements, showing exactly where bells are scheduled.
- **Standalone Bell Timer (`src/timer/StandaloneTimer.tsx`):**
  - Tiny breathing-circle visualizer representing meditation breath pacing.
  - Picking a bell, setting total duration (e.g., 20 mins), and setting an interval (e.g., every 5 mins).
  - Silent audio path except for the periodic bell rings.
  - Lives at route `/timer`.

### 6. Module Plan

We will structure the new modules as follows:
```
src/audio/bells/
  registry.ts             // Bell bank definition (ids, names, files, licenses, descriptions)
  loader.ts               // BellLoader (loads, decodes, and caches AudioBuffers lazily)
  scheduler.ts            // BellScheduler (manages scheduling events against AudioContext clock)
src/listening/
  BellScheduleEditor.tsx  // UI for editing the bell schedule
src/timer/
  StandaloneTimer.tsx     // The breathing visualizer + silent meditation bell timer
public/bells/             // Shipped Opus files
```

## Verification Plan

### Automated Tests
- **CI Bell Licenses Check:** Verify that all bells declared in the registry are documented in `LICENSES.md` and have a clean license string.
- **Bell Decoder Test:** Test that all Opus assets decode successfully using a headless Web Audio decode test.
- **Scheduler Test:** Unit test for `BellScheduler` utilizing a mock clock to verify sample-accurate scheduling.
- **Migration & URL Test:** Ensure legacy boolean flags (`opening_tone`, `closing_tone`) correctly migrate to `bell_schedule` on URL boot and database load.

### Manual Verification
- Verify preview play button sounds crisp, dry, and present.
- Schedule recurring bells (e.g., every 1 min) in a listening session and confirm they fire with sample-accuracy and do not go through the main reverb.
- Open Standalone Timer at `/timer`, start a session with a 10s interval, verify the breathing circle flows smoothly and the bell sounds at 10s intervals.
- Verify that deterministic renders produce matching bell timestamps.
