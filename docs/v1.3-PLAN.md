# Anneal Ambiance v1.3 Auth + Claim Plan

This document details the architecture, data models, API surface, security model, and implementation details for adding real user accounts (Magic Link + Google + GitHub OAuth) and content claim flow to Anneal Ambiance, keeping the anonymous-first substrate intact.

---

## 1. Final Data Model

To support case-insensitive emails in PostgreSQL while maintaining portability with SQLite (used for local/CI tests), we use standard `sa.String()` in SQLAlchemy decorated with a variant for PostgreSQL `CITEXT`. In the Alembic migration, we conditionally create the `citext` extension if running on PostgreSQL.

### SQL Schemas (PostgreSQL Representation)

```sql
-- Conditionally run: CREATE EXTENSION IF NOT EXISTS citext;

CREATE TABLE accounts (
  id              UUID PRIMARY KEY,
  email           CITEXT UNIQUE NOT NULL,    -- case-insensitive, verified email is mandatory
  email_verified  BOOLEAN NOT NULL DEFAULT FALSE,
  display_name    TEXT,                      -- optional, public
  avatar_seed     TEXT,                      -- deterministic phase-portrait seed (caching/overrides)
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  last_login_at   TIMESTAMPTZ
);
CREATE INDEX idx_accounts_email ON accounts(email);

CREATE TABLE account_providers (
  account_id      UUID NOT NULL REFERENCES accounts(id) ON DELETE CASCADE,
  provider        TEXT NOT NULL,             -- 'email' | 'google' | 'github'
  subject         TEXT NOT NULL,             -- provider-specific user ID or verified email
  PRIMARY KEY (provider, subject)
);
CREATE INDEX idx_account_providers_account ON account_providers(account_id);

-- Add parent relationship to users (which holds anon IDs)
ALTER TABLE users
  ADD COLUMN account_id UUID REFERENCES accounts(id) ON DELETE SET NULL;
CREATE INDEX idx_users_account ON users(account_id) WHERE account_id IS NOT NULL;

CREATE TABLE sessions (
  id              UUID PRIMARY KEY,          -- random opaque UUID
  account_id      UUID NOT NULL REFERENCES accounts(id) ON DELETE CASCADE,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  last_seen_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  expires_at      TIMESTAMPTZ NOT NULL,
  user_agent      TEXT,
  ip_hash         TEXT                       -- SHA-256 hashed IP for rate-limiting & security checks
);
CREATE INDEX idx_sessions_account ON sessions(account_id);
CREATE INDEX idx_sessions_expires ON sessions(expires_at);

CREATE TABLE magic_links (
  token           UUID PRIMARY KEY,          -- random one-time-use token
  email           CITEXT NOT NULL,
  intent          TEXT NOT NULL,             -- 'login' | 'signup' | 'add-email-to-account'
  account_id      UUID REFERENCES accounts(id) ON DELETE CASCADE,  -- nullable on signup
  expires_at      TIMESTAMPTZ NOT NULL,
  consumed_at     TIMESTAMPTZ
);
CREATE INDEX idx_magic_links_email ON magic_links(email);
CREATE INDEX idx_magic_links_token_unconsumed ON magic_links(token) WHERE consumed_at IS NULL;
```

## 2. Final API Surface

### 1. Auth Routing (`/api/v1/auth`)

- **`POST /api/v1/auth/email/request`**
  - **Purpose**: Generates and sends a magic link token to the specified email.
  - **Body**: `{ "email": "user@example.com", "intent": "login" | "signup" | "add-email-to-account" }`
  - **Response**: `202 Accepted` `{ "message": "If the email is valid, a link has been sent." }`
  - **Security**: Mitigates email enumeration by returning the exact same response status (202) and JSON structure regardless of whether the email exists. Gated by strict rate limits.
- **`GET /api/v1/auth/email/verify`**
  - **Purpose**: Consumes a magic link token, registers/logs in the user, sets an `am_session` HTTP-only cookie, and redirects the user back to the web application.
  - **Query Params**: `token=<uuid>`
  - **Response**: `302 Found` (Redirects to client app with `?signed_in=1`). If token is expired, invalid, or already consumed, redirects to client app with `?error=invalid_token`.
- **`GET /api/v1/auth/oauth/:provider/start`**
  - **Purpose**: Generates a secure OAuth state token, sets it as a transient HttpOnly cookie (`am_oauth_state`), and redirects the browser to the provider's authorize URL.
  - **Path Params**: `provider` (`google` or `github`)
  - **Response**: `302 Found` to Google/GitHub.
- **`GET /api/v1/auth/oauth/:provider/callback`**
  - **Purpose**: Exchanges the provider's authorization code for tokens, fetches user information, validates the state token cookie, logs in/registers the account, and issues the session.
  - **Path Params**: `provider` (`google` or `github`)
  - **Query Params**: `code=<str>`, `state=<str>`
  - **Response**: `302 Found` (Redirects to client app with `?signed_in=1`). If linking or state mismatch occurs, redirects with `?error=<code_str>`.
- **`POST /api/v1/auth/logout`**
  - **Purpose**: Revokes the current session from the database and clears the `am_session` cookie.
  - **Response**: `204 No Content` (Clears session cookie).
- **`GET /api/v1/auth/session`**
  - **Purpose**: Retrieves status and profile details of the currently authenticated account.
  - **Response**: `200 OK` `{ "account": { "id", "display_name", "avatar_seed", "email_verified" } }` or `{ "account": null }` if guest/anonymous.

### 2. Account Management Routing (`/api/v1/account`)

- **`GET /api/v1/account/me`**
  - **Purpose**: Returns the full private profile of the logged-in user. Gated to authenticated sessions only.
  - **Response**: `200 OK` `{ "id", "email", "display_name", "avatar_seed", "created_at", "last_login_at" }`.
- **`PATCH /api/v1/account/me`**
  - **Purpose**: Updates display name and/or avatar seed.
  - **Body**: `{ "display_name"?: "Alice", "avatar_seed"?: "seed_str" }`
  - **Validation**: Enforces display name structure (2-32 chars, alphanum + space/dash/underscore) and screens against profanity.
  - **Response**: `200 OK` Updated Account object.
- **`DELETE /api/v1/account/me`**
  - **Purpose**: Deletes the account immediately. Gated to authenticated sessions.
  - **Body**: `{ "confirm_email": "user@example.com" }`
  - **Response**: `204 No Content`. Cascades to disassociate all owned anon IDs (`users.account_id = NULL`), leaving existing public patches intact but orphaned.
- **`GET /api/v1/account/me/providers`**
  - **Purpose**: Lists linked credentials.
  - **Response**: `200 OK` `["email", "google"]`.
- **`DELETE /api/v1/account/me/providers/:provider`**
  - **Purpose**: Unlinks a credential provider from the active account.
  - **Validation**: Cannot delete the last remaining provider.
  - **Response**: `204 No Content`.

### 3. Claim Routing (`/api/v1/account/me/...`)

- **`POST /api/v1/account/me/claim`**
  - **Purpose**: Associates the active anonymous device ID with the logged-in user account.
  - **Body**: `{ "anon_id": "<uuid>" }`
  - **Validation**: Returns `409 Conflict` (`anon_id_already_claimed`) if the anon ID is already claimed by a different account.
  - **Response**: `200 OK` `{ "success": true }`.
- **`DELETE /api/v1/account/me/claim/:anon_id`**
  - **Purpose**: Revokes ownership of an anonymous ID (unclaiming it).
  - **Response**: `204 No Content`. Sets `users.account_id = NULL` for that ID.
- **`GET /api/v1/account/me/anon-ids`**
  - **Purpose**: Lists all claimed devices/anon IDs for the account, with content summaries.
  - **Response**: `200 OK` `[ { "anon_id", "patch_count", "capture_count", "recording_count", "source_count" } ]`.

### 4. Public Profiles (`/api/v1/profiles`)

- **`GET /api/v1/profiles/:account_id`**
  - **Purpose**: Public profile page metadata for rendering.
  - **Response**: `200 OK` `{ "id", "display_name", "avatar_seed", "created_at", "counts": { "patches": 5, "recordings": 2 } }`.

## 3. Final Authorization Model + Attack-Model

We represent identity on the server using a structured Request context state. The request is parsed by a unified auth middleware and injected into the FastAPI request state.

### Identity Context Structure

```python
class Identity:
    anon_id: uuid.UUID | None       # From x-anon-id header (active device ID)
    account_id: uuid.UUID | None    # From valid session cookie
    owned_anon_ids: list[uuid.UUID] # All anon IDs associated with this account_id (including active anon_id if claimed)
```

### Unified Ownership Verification Helper

To prevent "heuristic-drift," all endpoint handlers authorize operations using a single, central function:

```python
def is_owner(resource_owner_id: uuid.UUID, identity: Identity) -> bool:
    """Central ownership check used by patches, recordings, captures, and sources."""
    if identity.account_id is not None:
        # User is authenticated: does the resource owner match one of the user's claimed anon IDs?
        return resource_owner_id in identity.owned_anon_ids

    # Guest user: does the resource owner match the current device's anon ID?
    return identity.anon_id is not None and resource_owner_id == identity.anon_id
```

### Security & Attack Model

- **Session Hijacking**: Mitigated by issuing `HttpOnly`, `Secure` (in production), and `SameSite=Lax` session cookies. CSRF risks are handled by CORS configurations (no wildcards allowed when credentials are on) and standard Lax cookies preventing automatic inclusion on cross-site state-changing POST requests.
- **Anon ID Forgery / Hijacking**: In pure anonymous mode, access is gated by the `x-anon-id` header (which uses unguessable UUIDv4s). However, once a user **claims** an `anon_id`, that ID is permanently locked to their account! Any subsequent write or private read requested via `x-anon-id: claimed_uuid` **must** carry the corresponding account's session cookie; otherwise, the request is strictly rejected. This prevents hijackers from intercepting or forging anon IDs once they are claimed.
- **Magic Link Token Replay**: Magic link tokens are one-time use UUIDs. Verification checks that `consumed_at` is NULL and `expires_at > now()`, updating `consumed_at = now()` atomically on use. Replays fail instantly.

## 4. Magic Link Flow

- **Token Entropy & Expiry**: We use highly random `uuid.uuid4()` tokens (122 bits of entropy), set to expire exactly **15 minutes** after creation.
- **Email Delivery Service**: We select **Resend** because:
  - Resend provides a rich free tier of 3,000 emails/month (100/day limit), perfectly sizing our needs.
  - Cost per 1,000 emails is **$0**. Overages or expansion plans start at a flat $20/month for 50k emails (approx. $0.0004/email).
  - Its simple REST API makes integrations via `httpx` (already in our dependencies) incredibly clean, requiring zero external SDK packages.
- **Rate Limiting**:
  - Gated per-email: 5 requests per email per hour.
  - Gated per-IP: 20 requests per IP per hour.
  - Implemented in a unified Python RateLimiter to prevent abuse.

## 5. OAuth Providers

- **Google OAuth**:
  - **Redirect URL**: `https://accounts.google.com/o/oauth2/v2/auth`
  - **Params**: `client_id`, `redirect_uri`, `response_type=code`, `scope=openid email profile`, `state`
- **GitHub OAuth**:
  - **Redirect URL**: `https://github.com/login/oauth/authorize`
  - **Params**: `client_id`, `redirect_uri`, `scope=user:email`, `state`
- **Callback & State Cookie**:
  - A secure state token is generated client-side or server-side, placed into an HttpOnly cookie (`am_oauth_state`) with a 15-minute expiry.
  - The OAuth callback checks that the query param `state` matches the cookie `am_oauth_state`.
- **PKCE**: Deferred for v1.3. State cookie verification combined with backend-to-backend code exchange provides robust protection for our standard web structure.

## 6. Session Strategy

- **HttpOnly Cookie**:
  - Issued on success as `am_session = session_id`.
  - Flags: `HttpOnly`, `Secure` (in production, mapped from `ANON_COOKIE_SECURE`), `SameSite=Lax`. SameSite Lax is required to allow automated cookies to be sent when redirecting from Google/GitHub callbacks or magic links.
- **CSRF Strategy**:
  - Standard preflight CORS configuration rejects external API calls.
  - Additionally, state-changing endpoints (POST/PATCH/DELETE) require the `x-anon-id` or a custom client-side header (`x-request-id`), which browsers block cross-site unless preflight CORS allows it.
- **Session Lifespan & Renewal**:
  - Sessions last **30 days** from the last seen time.
  - Sliding-window renewal: If `last_seen_at` is older than 24 hours, the server automatically updates `last_seen_at` and pushes `expires_at` out by another 30 days.

## 7. Claim Mechanics

- **Manual vs. Auto-claim**: Manual claim is mandatory. If auto-claim was enabled, logging in on a shared/public computer would automatically seize that machine's local guest patches under the active account. Surfacing a banner ensures a deliberate user action.
- **Claim Conflict Handling**:
  - If a user attempts to claim an `anon_id` that is already bound to another account, the server raises `409 Conflict` with `anon_id_already_claimed`.
  - The client displays a descriptive warning: "This device is already claimed by another user account."

## 8. Display Name & Avatar

- **Name Validation**:
  - Length: 2 to 32 characters.
  - Format: Alphanumeric characters, spaces, hyphens, and underscores.
  - Screening: Checked against the static moderation wordlist.
- **On-Brand Deterministic Avatar**:
  - We generate an on-brand, custom SVG client-side using the `avatar_seed`.
  - The design features a dark background (`#0c0a09`) with amber waves (`#f59e0b`) calculated using a Lissajous curve or phase-portrait mathematical path derived from a hash of the seed. This looks incredibly premium, interactive, and fits the music theme perfectly.

## 16. Risks & Tradeoffs

- **Session Expiry Renewal Delay**: Updating `last_seen_at` on every API request would cause heavy DB write loads. Tradeoff: We only write session renewals once every 24 hours.
- **Profanity Filter False Positives**: Profanity filters might block legitimate display names. The display name is optional, and users can always change it later or fallback to guest "Anonymous" view.
