## 1. Session state machine

Added **into the existing `Orchestrator`** (the prompt: "orchestrator.ts gains
session-state machine + ArcRunner ownership"). The orchestrator's current
`start()`/`stop()` become internal steps the transitions call. React subscribes via a
new `useSession` hook (replacing `useAudioEngine.toggle`).

```
          ┌────────────────────────────────────────────────────────┐
          │                                                        ▼
  idle ──Begin──▶ starting ──(auto)──▶ running-open ──Stop──▶ stopping ──(auto)──▶ idle
                      │                                          ▲
                      └──(auto, mode=arc)──▶ running-arc ────────┤
                                                  │  Stop ───────┘
                                                  └─ arc done (orchestrator) ─┘
```

| State          | Trigger in                                          | Effect on the audio graph                                                                                                                                                                                                                            |
| -------------- | --------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `idle`         | boot / `stopping`→auto                              | No ctx/graph (orchestrator instance may exist; `start()` not called). UI shows `Begin` / `Begin · MM:SS`.                                                                                                                                            |
| `starting`     | user `startSession()` from `idle`                   | Snapshot start params; call existing `this.start()` (builds ctx + post-fx + active engine; 3 s bus fade-in begins). If arc: build `ArcRunner`, record `arcT0 = ctx.currentTime`, start the 20 Hz arc tick. Near-instant → transitions out same turn. |
| `running-open` | `starting`→auto (mode=open)                         | Existing behavior: drift loop runs, sculpt + engine-swap live. No arc tick.                                                                                                                                                                          |
| `running-arc`  | `starting`→auto (mode=arc)                          | Arc tick drives params at 20 Hz (§5). Drift loop also runs. Sculpt + engine selector locked (§6).                                                                                                                                                    |
| `stopping`     | user `stopSession()`, or orchestrator on arc `done` | The home for the fade-out. **Arc-end:** ramp `masterVol.gain` → 0 over **4 s** (`RETURNING` label), then teardown. **Manual stop:** existing `this.stop()` path (per-engine bus fade `FADE_OUT_TC` + `TEARDOWN_MS`). Arc tick + drift cleared.       |

**Who triggers each transition**

- `idle → starting`: **user** (Begin → `startSession`).
- `starting → running-open|running-arc`: **orchestrator** (auto, by `mode`).
- `running-* → stopping`: **user** (Stop/Settle → `stopSession`) **or** **orchestrator** (only from `running-arc`, when `ArcRunner.tick()` returns `done`).
- `stopping → idle`: **orchestrator** (auto, after fade + teardown resolve).

**Illegal transitions — decision.** Per the prompt ("illegal transitions are bugs"),
an internal `transition(to)` helper **throws** `IllegalTransitionError` on a truly
invalid edge. The public API stays crash-proof for user mashing: `startSession()` is
a **no-op + `console.warn`** unless `idle`; `stopSession()` is a **no-op** if already
`stopping`/`idle`. (Mirrors the orchestrator's existing `if (this.running) return`
guard in `start()`.)

**Engine-swap during arc:** `setEngine()` is **rejected (no-op + warn)** while
`running-arc` — changing engines mid-arc is out of scope (deferred list). Allowed in
`running-open` (unchanged v0.3 behavior).

## 2. Arc data model (finalized)

`src/session/types.ts`:

```ts
export type SessionMode = 'open' | 'arc';
export type SessionState =
  | 'idle'
  | 'starting'
  | 'running-open'
  | 'running-arc'
  | 'stopping';

export type CurveName = 'linear' | 'easeInOut' | 'exponential';

/** Arc-targetable keys = sculptable shared params, excluding volume. */
export type ArcTargetKey =
  | 'rootFreq'
  | 'spread'
  | 'density'
  | 'coupling'
  | 'drift'
  | 'brightness'
  | 'space';

export type TargetValue = number | 'min' | 'max'; // multiplier-on-start, or bound
export type SegmentTargets =
  | Partial<Record<ArcTargetKey, TargetValue>>
  | 'restoreStart';

export interface ArcSegment {
  fraction: number;
  curve: CurveName;
  targets: SegmentTargets;
}
export interface Arc {
  id: string;
  name: string;
  description: string;
  segments: ArcSegment[];
}
```

`ArcRunner` consumes the **existing** `AnnealEngineCapabilities` (only
`densityLockedWhilePlaying` matters) — no new `Capabilities` type.

### Target value semantics — **confirmed**, with a precise resolution model

The "multiplier on the user's starting value" approach is **kept** — pre-session
sculpting (and any FM engine-params) becomes the neutral pose; one arc deforms any
patch. Resolved **once at construction** so `tick()` is pure arithmetic:

- **numeric `v`** → `clampParam(key, startParams[key] * v)`.
- **`'min'`/`'max'`** → the param's declared bound (from `BOUNDS` in `params.ts`).
- **`'restoreStart'`** → captured `startParams[key]` for every key in the arc's active set.

**Active key set** = union of keys any segment targets. **Anchor chain** (precomputed
per segment, per active key):

- `start[0][k] = startParams[k]` (multiplier 1.0).
- `end[i][k]   = resolve(segment[i].targets, k)`, multiplier always on `startParams[k]`
  (not the previous anchor). A key absent from segment `i` holds: `end[i][k] = start[i][k]`.
- `start[i+1][k] = end[i][k]` (continuity, no jumps).

Within segment `i`, `tick` lerps `start[i] → end[i]` by the eased local progress. This
makes Bell's dwell a flat hold and `restoreStart` a smooth return to the captured pose. ✔

**Scope note:** v0.4 arcs target only the **shared** params (`ArcTargetKey`). Engine
params (e.g. `fm.modIndex`) are **not** arc-targeted in v0.4 — flagged as a clean
extension point for user-defined arcs (§10).

## 3. Final preset arcs (`src/session/arcs.ts`)

Numeric values are **multipliers on the user's start value** (then clamped).

### `bell` — "Bell Curve" · _Open, deepen, return._

| seg    | fraction | curve     | targets                                            | why                                                                                                           |
| ------ | -------- | --------- | -------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| settle | 0.33     | easeInOut | `rootFreq 0.7, coupling 1.3, drift 0.6, space 1.4` | Lower register (calmer), tighter coupling (more unison/consonance), quieter drift (stillness), deeper reverb. |
| dwell  | 0.34     | linear    | `rootFreq 0.7, coupling 1.3, drift 0.8, space 1.4` | Hold near the floor; drift nudged up vs. settle-end so the deep state still breathes.                         |
| return | 0.33     | easeInOut | `restoreStart`                                     | Ease all four back to the captured pose.                                                                      |

Active keys `rootFreq, coupling, drift, space` — none is density, all live on both
engines → **Bell runs fully** in v0.4.

### `dawn` — "Dawn" · _Sparse and dark, gradually opening._ · 1 seg, `easeInOut`: `brightness 1.6, density 'max', spread 1.05`

### `dusk` — "Dusk" · _Bright and open, gradually closing._ · 1 seg, `easeInOut`: `brightness 0.4, density 'min', spread 0.95`

> **Caveat (§0, §9):** both engines lock density while playing → the `density` target

> Flagging for explicit sign-off.

## 5. Interpolation contract

**`ArcRunner` is pure** — no `Date`, no `performance.now`, no `setInterval` inside.
All time is the `elapsedSec` argument.

```ts
new ArcRunner(arc, durationSec, startParams /*AnnealMusicParams*/, caps /*AnnealEngineCapabilities*/)
tick(elapsedSec): { params: Partial<AnnealMusicParams>; segmentIndex: number; progress: number; done: boolean }
```

- **Construction:** validate fractions; resolve sentinels + anchor chain (§2); if
  `caps.densityLockedWhilePlaying` and any segment targets `density`, drop `density`
  from the active set and push a warning to `runner.warnings`. After this, `tick`
  touches no sentinels.
- **Tick rate: 20 Hz (50 ms)** — matches `DRIFT_INTERVAL_MS`, so arc writes and drift
  updates share a cadence (no beating between the two loops).
- **Clock source:** the **orchestrator** owns the `AudioContext`, so the arc tick (a
  `setInterval`) reads `this.ctx.currentTime − arcT0` each fire and passes _that_ to
  `tick()`. `setInterval` jitter never accumulates — long arcs land on the audio clock.
- **Per tick:** `progress = clamp(elapsed/durationSec,0,1)`; locate segment by
  cumulative `fraction*durationSec`; `localT`; `eased = curve(localT)`; per active key
  `lerp(start[i][k], end[i][k], eased)`.
- **Done:** `elapsed ≥ durationSec` → final anchor, `done:true`, `progress:1`. Idempotent past the end.
- **Divergence (store vs. arc):** can't happen with controls locked, but spelled out —
  the arc is **authoritative and store-independent**: every tick recomputes absolute
  values from the _construction-time_ anchors, never from current store values, so a
  stray write can't compound; the next tick overwrites it.

**Data flow per tick:** orchestrator's arc tick → injected `applyArcParams(partial)`
writer (wired by `useSession` to `useParamStore.getState().setMany`) → existing
store→effect→`orchestrator.setSharedParams` channel updates `this.shared` (so the
drift loop sees new coupling/drift) + ramps post-fx, and the locked sliders re-render
to show live values. (Injecting the writer keeps the orchestrator store-agnostic and
unit-testable with a spy. The ~20 Hz store churn is acceptable — sliders already
re-render on drift; flagged in §10.)

## 6. Locked-controls UX

During `running-arc`, every **sculptable** slider renders **disabled/greyed**,
reusing `ControlPanel.tsx`'s existing `disabled` branch (muted label + micro-cap +
dimmed value), but the slider `value` keeps tracking the store so the thumb **glides
as the arc interpolates** — the user sees the arc working. (Chosen over hiding/strip:
clearest "automated, here's what it's doing" signal, reuses existing markup.)

- The `locked` micro-cap copy switches to **`arc`** during a session.
- **`EngineSelector`** (exists) is disabled during arc; `setEngine` is also a no-op at
  the orchestrator (§1) as defense-in-depth. Engine-specific param sliders (FM) lock too.
- **Volume:** arcs never target `volume`. **Recommendation: leave volume live during
  arc** (listening comfort; same reasoning that keeps it out of the share URL). Flag if you'd rather lock it.

## 8. URL schema v3

- `SCHEMA_VERSION = 3`; `SUPPORTED_SCHEMA_VERSIONS = [1, 2, 3]`.
- **Encode** (extends `encodeState`): prepend `m=<open|arc>`; if arc, add `arc=<id>&dur=<seconds>`.
  Keeps the v2 `e=<id>` + shared + `fm.*` tail. Example:
  `#s=3:m=arc&arc=bell&dur=720&e=fm&rootFreq=110&…&fm.modRatio=2.00&…`
- **Session-selection state** (`mode`, `arcId`, `durationSec`) is added to the **param
  store** (consistent with how `engineId`/`engineParams` already live there), so the
  existing `subscribeStoreToHash` writer picks it up with no new wiring. (Alternative:
  a separate `session` store — rejected for now; one store = one URL writer.)
- **Decode** (extends `decodeState(version, payload)` → `DecodedState` gains `mode`,
  `arcId?`, `durationSec?`):
  - **v1 / v2** payloads → `mode:'open'`, ignore any `arc`/`dur` (consistent with how
    v1 already ignores `e=`/`fm.*`).
  - **v3 unknown `arc` id** → `mode:'open'` + warning → App shows toast **"Unknown arc, loaded open mode."**
  - **v3 `dur` out of bounds** → clamp to `[180, 3600]` s + warning.
  - **version > 3** → `readStateFromHash` returns `null` → existing "newer version" toast.
- `m`/`arc`/`dur` are consumed by the session branch of `decodeState` first, so they're
  not mis-flagged as "unknown keys."

## 9. Density-target handling for locked engines

- **Where:** `ArcRunner` constructor. If `caps.densityLockedWhilePlaying` and any
  segment targets `density`, drop it from the active set + record a warning
  (`"density target ignored: <engineId> locks density while playing"`).
- **UX surface — recommendation: console only (no toast).** Both engines lock density,
  so a toast would fire on _every_ Dawn/Dusk start — nagging for expected, unchangeable
  behavior. Orchestrator logs `runner.warnings` via `console.warn` at arc start.
  Optionally (low cost) the arc **summary line** shows a muted `· density held` note
  when the selected arc has a dropped density target on the active engine. I lean
  toward the inline note (it explains Dawn/Dusk's reduced drama). Flag if you'd prefer a toast or nothing.

## 10. Risks & tradeoffs

1. **Density-locked Dawn/Dusk (§3, §9).** Sweep brightness+spread only on both v0.4
   engines; resolved by an unlocked engine (granular, v0.9). Sign-off needed.
2. **Multiplier-only + shared-params-only model vs. user-defined arcs (v0.4+).** The
   timeline editor will likely want _absolute_ targets and _engine-param_ targets. The
   model extends cleanly: widen `TargetValue` to a tagged union (`{abs}`/`{mul}`/
   sentinels) and add an engine-param namespace to `ArcTargetKey`. No preset migration
   needed. Flagged, not built.
3. **`restoreStart` / partial-key segments.** Presets touch a consistent key set, so the
   anchor chain is unambiguous. User arcs whose segments touch _different_ key subsets
   rely on the "hold start until first targeted" rule (specified §2) — must be tested
   before the editor ships.
4. **20 Hz store churn.** Arc ticks call `setMany` 20×/s → React re-renders the control
   panel. Acceptable (drift already drives frequent renders; sliders are cheap). If it
   ever bites, the arc could push audio directly via `setSharedParams` and throttle the
   store write for display — noted, not done.
5. **AudioContext clock + backgrounded tabs.** Long-arc accuracy rides on
   `ctx.currentTime` (bypassing `setInterval` drift). When a tab is hidden some browsers
   suspend the context: `currentTime` stalls _and_ audio pauses — the arc effectively
   pauses with the sound and resumes on focus. No glitch, no silent over-run. Will
   document in `COMPAT.md` with tested durations.
6. **No pause/resume.** Stop abandons the arc, returning cleanly to idle from any state.
7. **Refactor blast radius.** Extending `Orchestrator` + swapping `useAudioEngine` for
   `useSession` touches `App.tsx`, `Visualizer` (progress overlay), `ControlPanel`
   (arc-lock), `EngineSelector` (lock). v0.2/v0.3 share + orchestrator + engine tests
   must stay green; the schema change is additive + backward-migrating, so old URLs keep loading.
