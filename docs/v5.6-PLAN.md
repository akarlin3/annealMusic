# AnnealMusic v5.6 · Experiment Framework · Plan

This document details the blueprint for the perceptual experiment framework in music cognition and MIR research for AnnealMusic v5.6.

---

## 1. Final `anneal.experiment` Python API

Within the sandboxed Pyodide environment, researchers define experiments by scripting with classes from the `anneal.experiment` sub-module.

```python
from anneal.experiment import Experiment, Stimulus, LikertResponse, ForcedChoice, Block, DemographicSurvey, FreeText, AdjustValue, ReactionTime, Continuous

# Instantiate experiment
exp = Experiment(
    title="Pitch height perception under detuning",
    description="Psychoacoustic evaluation of pitch stability.",
    consent_text="By participating you agree that your anonymized responses are collected...",
    debrief_text="Thank you. This study examines pitch detuning thresholds."
)

# Optional Demographics survey
exp.add_demographics(DemographicSurvey(fields=["age", "hearing_loss", "musical_experience"]))

# Define stimuli
stim_a = Stimulus(
    id="control",
    patch={"engine": "sine", "rootFreq": 220, "drift": 0.0},
    duration=4.0
)

stim_b = Stimulus(
    id="drifted",
    patch={"engine": "sine", "rootFreq": 220, "drift": 0.6},
    duration=4.0
)

# Add blocks of trials
exp.add_block(Block(
    name="main",
    trials=[
        {"stimulus": stim_a, "response": LikertResponse("How stable does this feel?", scale=7)},
        {"stimulus": stim_b, "response": LikertResponse("How stable does this feel?", scale=7)}
    ],
    randomize="full",
    counterbalance=True
))

exp.add_break(message="Take a short break. Click continue when ready.")

# Launches experiment preview runner on the JS thread
exp.run()
```

## 2. Built-in Response Type Implementations

We provide six interactive response components styled in a clinical, distraction-free aesthetic:

- **`LikertResponse(prompt, scale=N)`**: Displays $N$ numbered radio buttons or high-contrast buttons in a horizontal row.
- **`ForcedChoice(prompt, options=[...])`**: Renders a vertical layout of large select buttons or a drop-down list.
- **`FreeText(prompt, max_chars=500)`**: Displays a clean textarea with an interactive character countdown indicator.
- **`AdjustValue(prompt, range=(min, max), step=val, target_param='drift')`**: Renders a horizontal slider. Moving the slider dynamically updates the synth parameter on the active audio engine in real time so the subject can adjust it to match their perception.
- **`ReactionTime(prompt, target_key='Space')`**: Instructs the subject to press the `target_key` as soon as they perceive a stimulus change. Records the interval between stimulus onset and keypress in milliseconds using `performance.now()`.
- **`Continuous(prompt, duration, scale)`**: Displays a vertical or horizontal slider. The participant continuously moves it during stimulus playback. The position of the slider is sampled at **30Hz** (every ~33ms) over the trial duration.

## 5. Randomization & Williams Latin Square Counterbalancing

We support three randomization modes:

- **`'fixed'`**: Stimuli are presented exactly in the order defined in python.
- **`'full'`**: Complete independent random shuffling of all trials in the block.
- **`'within-block'`**: Trials within each block are shuffled, keeping the block order fixed.

### Counterbalancing Algorithm (Williams Latin Square)

If `counterbalance=True`, trials within a block are counterbalanced to eliminate order-carryover effects. We generate a Williams Latin Square row based on the subject's ID hash:

```typescript
export function getLatinSquareRow(N: number, subjectIndex: number): number[] {
  const row: number[] = [];
  const shift = subjectIndex % N;

  const baseSequence: number[] = [];
  for (let i = 0; i < N; i++) {
    if (i % 2 === 0) {
      baseSequence.push(i / 2);
    } else {
      baseSequence.push(N - (i + 1) / 2);
    }
  }

  for (let i = 0; i < N; i++) {
    row.push((baseSequence[i] + shift) % N);
  }

  return row;
}
```

## 6. Timing Precision Documentation

- **Sound Playback:** Sub-millisecond timing accuracy. AudioContext clocks anchor the sample-level rendering pipeline.
- **Response Capture:** 5–15ms latency due to browser UI thread cycles and hardware event loops. Response times are captured using high-precision `performance.now()`.
- **Limitation Statement:** AnnealMusic is designed for behavioral cognition studies (Likert scaling, categorization, rating). It is NOT recommended for sub-10ms EEG, MEG, or ERP phase-locking experiments.

## 7. IRB-Friendly Defaults Checklist

- [x] **Explicit Consent Required:** The experiment start button remains locked until the subject checks the consent checkbox.
- [x] **"Withdraw" Command Visible:** An ever-present "Withdraw" button in the corner lets subjects abort the session. It triggers a clean-up routine offering to discard all temporary logs or download partial records before deleting.
- [x] **No Hidden Tracking:** Zero cookies, zero local storage tracking, zero third-party calls during execution.
- [x] **Transparency Preview:** Displays the exact structured data captured before the subject downloads or POSTs.

## 9. Risks & Mitigations

1. **Audio Synthesis Glitches on Low-End Devices:**
   - _Risk:_ Heavy patches might stutter, confounding reaction times.
   - _Mitigation:_ Audio state is monitored. If high processing latency is detected, a warning is added to the trial's metadata record.
2. **Subject Accidental Browser Refresh:**
   - _Risk:_ Tab close or refresh wipes all active trial variables.
   - _Mitigation:_ We add standard `beforeunload` event listeners warning the participant that refreshing will abort the session.
3. **Institutional IRB Varied Policies:**
   - _Risk:_ Some institutions require active server persistence rather than ZIP downloads.
   - _Mitigation:_ Provide both local ZIP downloads AND standard secure HTTP POST integrations.
